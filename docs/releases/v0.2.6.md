# 神笔写作 v0.2.6 更新日志

发布日期：2026-05-10

## 版本亮点

本次更新是 `0.2.5 -> 0.2.6` 的一次桌面端工作区体验与 Agent 稳定性更新。重点包括电脑端系统文件资源管理器入口、SQLite 书籍工作区的桌面镜像同步、目录栏工具简化、网文大白话写作规则内置，以及 AI 流式请求失败后的自动续跑。

## 重点更新

### 1. 电脑端系统文件资源管理器入口

- 图书编辑页面的目录栏顶部新增“在系统文件资源管理器中打开书籍文件夹”按钮。
- 由于当前书籍工作区以 SQLite 存储为事实源，电脑端会先把当前书籍导出为应用数据目录下的真实镜像文件夹，再用系统文件资源管理器打开。
- 作者可以在系统资源管理器里直接新增、删除、重命名或修改文件，适合批量整理章节、设定、素材和项目文件。
- 移动端保持原有底部导航与数据库工作区，不显示该桌面专用入口。

### 2. SQLite 工作区与桌面镜像同步

- 新增桌面镜像模块，将数据库中的 `book_workspace_entries` 导出成真实目录结构。
- 点击目录栏“刷新当前书籍”时，会先把镜像文件夹中的外部改动同步回 SQLite，再刷新应用内文件树。
- 同步支持新增文件、新增文件夹、删除文件、删除文件夹和内容修改。
- 如果编辑器里存在未保存草稿，刷新时会先导入外部镜像改动，再保存当前草稿，避免软件内正在编辑的内容被旧镜像覆盖。
- 镜像文件夹位于应用数据目录下的 `desktop-workspaces`，仍由 SQLite 工作区保持跨平台主事实源。

### 3. 图书目录栏工具简化

- 移除目录栏顶部的“展开全部文件夹 / 折叠全部文件夹”按钮。
- 保留刷新、新建文件夹、新建文件等高频操作，减少顶部工具栏拥挤。
- 已更新桌面端和移动端相关测试，确认旧按钮不再出现。

### 4. 内置网文大白话写作规则

- 主代理配置新增 `Web Novel Plain Style Rules`。
- 正文、续写、改写、润色、去 AI 味任务默认遵守大白话规则。
- 规则强调普通用词、短句、频繁换行、简单标点、直白情绪、生活化动作、真人对话和避开机器味套话。
- 该规则只约束写作表达风格，不强行干预剧情结构。

### 5. Agent 流式请求自动续跑

- Agent 核心循环新增 AI 请求失败自动续跑机制。
- 遇到流式响应中断、响应体解码失败、`fetch failed`、空输出等短暂错误时，系统会自动追加“继续执行”提示并续跑。
- 连续失败计数达到 5 次后停止自动续跑，并抛出包含模型、Base URL、失败次数、错误信息和已生成片段数的详细报告。
- 中止运行仍按用户主动取消处理，不会误触发自动续跑。

### 6. 版本与更新检测

- 项目版本更新到 `0.2.6`。
- 同步更新 `package.json`、`package-lock.json`、`Cargo.toml`、`Cargo.lock` 和 `tauri.conf.json`。
- README 的版本徽章和下载链接改为 `0.2.6`。
- 更新检测相关测试改为以 `0.2.6 -> 0.2.7` 作为新版本场景。

## 兼容与行为变化

- 书籍数据仍以 SQLite 为跨平台事实源。
- 电脑端资源管理器打开的是应用生成的镜像文件夹。
- 外部文件改动需要点击“刷新当前书籍”后同步回应用内工作区。
- 镜像同步会重建当前书籍的数据库条目，建议避免在同一时间同时大量修改镜像文件夹和应用内多个文件。
- `opener:default` 权限仍不开放任意前端 `openPath`，打开文件夹通过受控 Rust 命令完成。
