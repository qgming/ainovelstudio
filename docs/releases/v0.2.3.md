# 神笔写作 v0.2.3 更新日志

发布日期：2026-04-27

## 版本亮点

本次更新围绕「创作台提示词模板可编辑化」「模型推理模式配置扩展」和「项目上下文补全」三条主线推进，同时完成 `0.2.3` 双端发版收口。整体目标是让创作台的 AI 指令更可控、模型供应商配置更细致，并让工作区在读取 `AGENTS.md` 之外，稳定纳入 `README.md` 作为补充上下文。

## 重点更新

### 1. 创作台支持按动作自定义提示词模板

- 创作台 6 个常用动作现在都支持单独编辑提示词模板：
  - 批量生成细纲
  - 批量生成设定
  - 更新设定
  - 章节写作
  - 设定更新
  - 自由输入
- 页面侧的动作按钮新增提示词编辑入口，并在已自定义时显示醒目标记，方便区分默认模板和工作区专属模板
- 提示词模板拆分为：
  - 代码硬编码的动态头部
  - 用户可编辑的指令主体
- 这样既能保证当前目标、当前文件、章节细纲等运行时变量稳定注入，也允许用户长期微调自己的创作提示词风格

### 2. 自定义提示词模板支持持久化与重置

- Rust 端新增扩写提示词模板存储表 `expansion_prompt_templates`
- 新增 Tauri 命令：
  - `list_expansion_prompt_templates`
  - `save_expansion_prompt_template`
  - `reset_expansion_prompt_template`
- 提示词模板按 `workspaceId + actionId` 隔离保存，不同创作项目之间互不干扰
- 当数据库中没有自定义记录时，前端会自动回退到内置默认模板
- 对话框中支持一键恢复默认值，方便随时撤回过度定制

### 3. 创作台提示词构建逻辑收口

- 创作台原本分散在页面中的多段提示词拼装逻辑被抽到独立模板模块
- `composePrompt` 负责统一拼接“动态头部 + 指令主体”，减少页面内联字符串的维护负担
- 批量细纲、批量设定、章节写作、设定更新和自由输入全部走同一套模板机制
- 相关测试同步改为围绕模板主体和头部拼装进行断言，后续继续扩展动作类型时成本更低

### 4. 模型设置支持推理模式配置

- 模型供应商配置新增 `reasoning_effort` 支持，可配置：
  - `low`
  - `medium`
  - `high`
  - `xhigh`
- 前端模型设置页补充了对应配置项和合法性校验
- Agent 配置标准化逻辑会统一处理推理模式字段，避免供应商配置格式漂移
- 请求发送链路已把推理模式透传到 provider 侧，后续接入支持该字段的模型时可以直接生效

### 5. 项目上下文补齐 README 读取链路

- 扩写与工作区相关链路在读取 `project/AGENTS.md` 的同时，现已把 `project/README.md` 纳入上下文
- 章节写作、设定生成、批量细纲等提示词会更稳定读取项目说明、题材方向和额外规则
- 相关测试也同步覆盖 README 的存在与读取行为，减少后续回归

### 6. 版本号与发版配置同步到 0.2.3

- 项目版本号由 `0.2.2` 升级到 `0.2.3`
- 已同步更新：
  - `package.json`
  - `package-lock.json`
  - `src-tauri/Cargo.toml`
  - `src-tauri/Cargo.lock`
  - `src-tauri/tauri.conf.json`
  - `README.md`
- 关于页、侧边栏、更新 store 与 App 中的版本测试断言已同步切换到：
  - 当前版本 `0.2.3`
  - 可用更新样例 `0.2.4`
