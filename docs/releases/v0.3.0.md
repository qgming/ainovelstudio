# 神笔写作 v0.3.0 更新日志

发布日期：2026-05-21

## 版本亮点

本次更新是 `0.2.9 -> 0.3.0` 的一次较大版本收口，重点围绕三条主线推进：

- Agent 执行模型继续从“模式控制”收敛到更明确的 YOLO / 计划 / 工具协作链路
- 图书工作区三栏体验继续打磨，尤其是 Agent 区、编辑区和默认打开行为
- 设置、调试、排行榜、技能库等外围页面完成新一轮结构整理与界面升级

除了功能扩展，这个版本还处理了一批长期影响体验的细节问题，包括写入锁冲突、运行时提示词魔法过多、消息区噪音偏重，以及工作区初始空白页的问题。

## 重点更新

### 1. Agent 执行链路继续收敛，YOLO / 计划 / 控制工具更清晰

- 继续推进 autopilot / YOLO 执行链路，围绕“目标 -> 执行 -> 验证 -> 继续”的循环做收口
- 原有 mode control / workflow control / task tool 等旧分支进一步移除，减少概念重叠
- 计划工具、控制工具和运行循环继续整理，工具输出结构和计划状态表达更统一
- 模型能力、推理强度、输出截断等底层能力补齐测试与类型约束

### 2. Agent 面板界面重做，消息区更轻、更安静

- Assistant 正式回复改成更接近正文的纯文本阅读样式，减少卡片感
- 工具消息、思考消息、任务进度等折叠态改成更轻的单行信息
- 展开态卡片背景进一步淡化，保留状态信息但降低视觉干扰
- 输入区上方新增更一致的任务面板样式，支持紧凑展开 / 收起，并与输入框边框做一体化衔接
- 输入区任务面板、Agent 组件图标和展开行为进一步清理，减少重复箭头和装饰性噪音

### 3. 移除一批内置提示词魔法，减少隐式行为

- 移除 turn prompt 中按“创作/续写”“分析/诊断”等标签自动触发的内置特殊提示逻辑
- 运行时上下文中不再固定注入“当前激活文件”等多余提示信息
- Prompt 相关测试同步更新，Agent 现在更偏向显式上下文与明确工具链，而不是关键词魔法

### 4. 图书编辑区布局和默认打开行为优化

- 中间编辑区的文件名、字数和常用操作已移动到顶部工具栏，减少底部干扰
- 编辑页顶部工具栏圆角与整体卡片样式统一，界面层次更稳定
- 进入图书编辑页时，如果当前没有有效打开文件，会默认打开 `.project/README.md`
- 这样首次进入工作区或恢复工作区时，不再出现“中间空白什么都没打开”的状态

### 5. 修复工作区写入锁问题，提升稳定性

- 修复书籍工作区写入时偶发的 `database is locked` 问题
- SQLite 写入事务行为调整为更适合当前工作区写回链路的模式
- 对连续保存、工具写回、自动执行等高频写操作场景更友好

### 6. 设置页、调试能力和模型配置继续增强

- 新增独立调试模块与调试页面能力，便于排查运行时与数据问题
- 设置页结构继续调整，模型提供商、目录、备份、默认 Agent 等模块边界更清晰
- 模型配置、目录按钮、Provider 卡片、关于页和更新说明弹窗都做了同步优化
- 用量统计与日志展示继续重构，信息组织更规整

### 7. 排行榜、技能库与通用页面继续升级

- 新增和强化排行榜 / 统计相关后端与前端能力
- 排行榜页面、统计页面、技能列表、技能详情页做了新一轮结构和 UI 升级
- 顶部标题、返回标题、移动端导航、PageShell 等通用基础组件继续完善
- 主题、页面结构和移动端适配也做了补强

### 8. 发布流程与双端构建产物同步到 0.3.0

- 项目版本已更新到 `0.3.0`
- 本地已完成 Windows EXE 与 Android arm64 APK 构建
- 新增基于 GitHub Actions 的自动发布 workflow：推送 `v*` tag 后可自动构建 exe / apk 并创建 GitHub Release
- `app.json` 继续保留为手动上传到更新服务器的清单文件，不混入 GitHub Release 资产

## 兼容与行为变化

- Agent 不再依赖一组内置标签式提示词来切换特殊行为，更多依赖显式上下文、技能和工具调用
- 进入工作区时的默认打开文件行为变为优先打开 `.project/README.md`
- GitHub Release 自动发布现在依赖 tag 触发 workflow，后续建议按 `v版本号` 的方式发版
