# 神笔写作 v0.2.4 更新日志

发布日期：2026-04-28

## 版本亮点

本次更新聚焦「工作流判断节点容错」「模型推理开关默认安全值」和「会话上下文面板重构」三条主线，同时完成 `0.2.4` 双端发版收口。整体目标是让工作流在真实创作场景下更稳、模型设置页更安全、Agent 会话上下文展示更贴近实际调用成本。

## 重点更新

### 1. 工作流判断节点补齐结构化判定兜底解析

- 判断节点原先强依赖 `workflow_decision` 工具提交结构化结果
- 当模型直接输出文本结论、漏掉工具调用时，流程会中断并提示缺少结构化判定结果
- 现在新增回退解析能力，支持两类正文输出直接恢复结构化判定：
  - JSON / JSON 代码块
  - 标签式文本，如“通过结论 / 判断原因 / 问题列表 / 修订摘要”
- 这样即使模型没有显式调用工具，只要结论内容完整，工作流也能继续执行

### 2. 模型设置页的 reasoning_effort 默认关闭

- 模型设置页的“思考模式 reasoning_effort”现在默认以关闭态展示
- 关闭时会明确显示“已关闭”，并禁用强度按钮，避免视觉上误判为已启用
- 切换模型、应用推荐供应商、应用预存供应商时，会自动把 `enableReasoningEffort` 复位为 `false`
- 这能减少把不兼容的 reasoning 参数带到新模型上的概率

### 3. Agent 会话上下文面板改为会话级指标视图

- Agent 面板顶部的上下文入口从“工作区上下文”切换为“会话上下文”
- 新面板围绕当前会话展示：
  - 会话标题
  - 消息数
  - 当前模型
  - 模型调用次数
  - 本轮 token
  - 会话累计 token
  - 推理 token
  - 缓存读写 token
- 同时增加上下文拆分可视化，帮助更快理解最近一次模型调用的成本构成

### 4. BookAgentPanel 收口到会话上下文入口

- 侧边 Agent 面板移除了原先偏工作区信息的上下文展示逻辑
- 新入口更聚焦当前对话过程本身，减少工作区级信息和会话级信息的混淆
- 相关渲染与交互测试同步调整

### 5. 版本号与发版配置同步到 0.2.4
