# 神笔写作 v0.2.5 更新日志

发布日期：2026-05-09

## 版本亮点

本次更新是 `0.2.4 -> 0.2.5` 的一次 Agent 核心能力重构与双端发版收口。重点包括长篇写作工作流、YOLO 自动执行模式、线性上下文与临时子任务、项目 Canon 事实源、Tauri 流式模型代理，以及刚完成的流式事件和 `todo` 工具稳定性修复。

## 重点更新

### 1. 长篇写作工作流与项目 Canon

- 新增长篇写作相关的工作区模板、上下文清单和 Canon 事实源能力
- Agent 能读取项目状态、章节线索、人物与设定边界，并围绕长篇创作做更稳定的上下文注入
- 新增 `canon_query` 工具，用于查询 `.project/canon`、角色、地点、伏笔、能力边界和章节线索
- 新增 Canon delta 解析与测试，支持把运行中的事实变化沉淀回项目结构

### 2. Agent 运行模式收敛与 YOLO 自动执行

- Agent 模式收敛为更清晰的手动协作与自动执行两类路径
- 新增 YOLO / autopilot 模式，用于围绕目标连续推进、自动检查和循环执行
- Composer 展示当前计划状态、自动执行目标和运行提醒，减少长任务中断感
- coaching 提示词重写，强调当前执行状态、真实阻塞和下一步推进

### 3. 线性上下文、任务记忆与临时 subagent

- 新增线性会话上下文组装，降低长对话历史对后续轮次的噪音
- 新增任务记忆压缩，用于保留目标、进展、事实、约束、路径和工具使用记录
- 新增 `task` 工具，可按需创建临时 subagent，在隔离上下文里完成局部任务
- Agent 工具循环拆分为更清晰的 core loop、session、part queue 和 stream part 映射

### 4. 聊天运行时与持久化重构

- `chatRun` 运行时拆分为 message runner、session factory、terminal handlers、runtime controller 等模块
- 用户消息、助手占位消息、流式片段和终止状态的持久化路径更集中
- 停止运行、工具请求、pending ask、自动续跑和 compact 阈值处理更明确
- Agent 消息列表新增长会话渲染优化：默认只渲染最近 80 条消息，并折叠更早消息

### 5. 模型代理、流式输出与更新能力

- Tauri 新增 provider forward / stream proxy，前端模型请求通过本地安全代理转发
- Agent 流式请求支持 start / chunk / end 事件桥接，Windows updater 产物继续生成签名和 `latest.json`
- 模型连接测试继续使用非流式短请求，Agent 对话使用 AI SDK `streamText` 流式链路
- 修复 `provider-stream` 事件字段命名不一致导致 Agent 一直显示“正在思考”的问题
- 前端同时兼容 `requestId` 和旧版 `request_id`，降低旧包或边界事件导致的卡死风险

### 6. `todo` 工具稳定性修复

- 修复模型偶发把计划参数写成 `todos` 字段的问题
- 修复模型偶发把计划数组序列化成字符串导致 schema 校验失败的问题
- `todo` 的 AI SDK schema 和本地执行器均兼容 `items` / `todos`，并自动解析字符串化 JSON 数组
- 系统提示和工具描述明确要求使用 `items` 数组，减少后续错误调用概率

### 7. 工作区与架构整理

- 移除旧的 Expansion、内置 Agent 与 Workflow 页面和后端模块，降低维护面
- 前端按业务域重组为 agent / books / settings / skills / update / shared 等目录
- Rust 侧按 app / domains / infrastructure 分层，book workspace、chat、data sync、skills、usage 等模块边界更清晰
- 书籍工作区树新增自然排序，章节和文件名排序更符合阅读直觉
- 新增 Skeleton、Spinner、Switch、Table、Tabs、Textarea、Tooltip、Toaster 等 UI 基础组件

## 兼容与行为变化

- 旧 Expansion 页面、内置 Workflow 页面、内置 Agent 管理页已移除
- 长篇写作能力转向项目工作区、技能、工具和 Agent 模式组合
- `todo` 工具推荐参数为 `items`，但本版本兼容旧误用形态 `todos`
- Windows 自动更新清单 `latest.json` 指向 `v0.2.5` 的 Windows 安装包资产
