# 神笔写作 v0.2.9 更新日志

发布日期：2026-05-17

## 版本亮点

本次更新从 `0.2.8` 推进到 `0.2.9`，重点放在三个方向：让 Agent 会话和工作区文件之间的协作更顺手，让设置页的模型和用量信息更清楚，以及把用量统计从“临时从日志反推”升级为更可靠的独立数据结构。

这一版的目标不是堆新入口，而是把常用动作收进更自然的位置：历史会话的危险操作进入“更多”菜单，工作区文件可以直接加入 Agent 上下文，用量日志只保留最近明细，而长期累计和热力图由独立表保存。

## 重点更新

### 1. Agent 历史会话操作更克制

- 历史会话列表右侧新增“更多”按钮，不再把删除操作直接暴露在列表主视觉上。
- 删除会话移动到更多菜单中，降低误触概率。
- 当前会话仍保持不可删除，避免正在使用的对话被意外移除。
- 书籍工作区顶部的 Agent 历史下拉菜单也同步支持右侧更多操作，界面行为和独立历史面板保持一致。

### 2. 工作区文件可以直接加入 Agent 上下文

- 左侧工作区文件的更多菜单新增“添加到上下文”操作。
- 点击后会把该文件加入右侧 Agent 输入区上方的 `@文件` 上下文 chip，效果和输入区 `@` 按钮选择文件一致。
- 已加入上下文的文件会显示“已在上下文中”，避免重复添加。
- 桌面端如果 Agent 栏处于折叠状态，添加文件到上下文时会自动展开右侧 Agent 栏。
- 移动端添加文件到上下文后会切换到“助手”页，方便继续输入任务。
- Agent Composer 的手动上下文选择从本地组件状态提升到会话运行 store，`@` 按钮和文件树菜单现在共享同一份选择状态。

### 3. 工作区文件菜单补齐图标

- 文件夹更多菜单中的“新建文件夹”“新建文件”增加语义图标。
- 文件和文件夹共用的“重命名”“删除”增加图标。
- “添加到上下文”保留 `@` 图标，并和其他菜单项统一间距。
- 菜单仍保持紧凑工具型风格，便于快速扫读和重复操作。

### 4. 设置页导航顺序调整

- 设置页左侧导航中，“模型设置”已移动到“用量统计”上方。
- 新顺序更贴近实际使用路径：先配置模型，再查看模型调用带来的用量数据。
- 移动端设置列表同步使用同一份导航配置，因此排序也保持一致。

### 5. 用量统计改为独立表存储

这一版对用量统计的数据结构做了重要调整：

- 新增 `usage_summary` 表，用于保存累计请求数、累计输入 token、输出 token、总 token、缓存命中、缓存写入和推理 token。
- 新增 `usage_daily_stats` 表，用于保存每日请求数和每日 token 总量，热力图直接读取该表。
- 新增 `usage_logs` 表，用于保存最近的用量明细日志。
- `usage_logs` 只物理保留最新 100 条，避免明细日志无限增长。
- 总请求数、总 token、热力图等长期统计不再从最近 100 条日志反推，因此不会因为日志裁剪而丢失累计数据。
- 首次读取或首次写入用量统计时，会从已有聊天消息中的 usage 信息回填独立表，兼容旧版本已有数据。
- 后续新的 Agent 模型调用会直接写入独立统计表和最近日志表。

### 6. 用量页面读取逻辑更清楚

- 统计卡片读取 `read_usage_summary`，展示长期累计值。
- 热力图读取 `read_usage_daily_stats`，展示每日聚合。
- 日志表读取 `read_usage_logs`，展示最近 100 条明细。
- 页面增加提示：日志只保留最新 100 条，累计指标单独保存。
- 时间范围筛选仍可作用于热力图和当前日志列表，但不改变顶部累计指标的长期总量含义。

### 7. 版本与发布信息同步到 0.2.9

- 项目版本已同步更新到 `0.2.9`。
- 同步更新 `package.json`、`package-lock.json`、`Cargo.toml`、`Cargo.lock` 和 `tauri.conf.json`。
- README 的版本徽章和下载链接已同步到 `0.2.9`。
- 关于页和应用壳层中的版本展示测试已同步更新。

## 数据结构变化

本次新增三张 SQLite 表：

- `usage_summary`
- `usage_daily_stats`
- `usage_logs`

其中 `usage_logs` 是短期明细表，只保留最近 100 条；`usage_summary` 和 `usage_daily_stats` 是长期统计表，不依赖日志表重新计算。
