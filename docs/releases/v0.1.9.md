# 神笔写作 v0.1.9 更新日志

发布日期：2026-04-22

## 版本亮点

本次更新围绕「项目状态结构化」「Agent 连续性记忆」「初始化体验」和「发布打包」做了集中优化。新建书籍后，工作区会更快进入可协作状态；Agent 读项目状态、维护 JSON 和承接工作流上下文也更稳定。

## 重点更新

### 1. 新建书籍初始化升级

- 默认项目目录由 `03_素材` 调整为 `03_规划`
- `03_规划` 用于承载参考资料、章节规划、大纲规划、命名备忘和阶段性方案
- 新建书籍时默认初始化 3 个状态 JSON：
  - `.project/status/project-state.json`
  - `.project/status/latest-plot.json`
  - `.project/status/character-state.json`
- `project-state.json` 增补默认文件索引、规划目录信息和章节规划推荐入口
- `.project/AGENTS.md` 的项目说明同步更新，默认把规划内容写入 `03_规划/`

### 2. JSON 工具增强

- `json` 工具新增 `ensure_template`
  - 用于按模板补齐缺失字段
  - 适合初始化旧项目状态文件或做结构升级
- `json` 工具新增 `history_append`
  - 用于向状态历史数组追加记录
  - 支持自动补 `updatedAt`
  - 支持限制历史条目最大保留数量
- `json` 工具新增 `patch`
  - 支持标准 JSON Patch 风格操作
  - 支持 `add / copy / move / remove / replace / test`
- `json.batch` 校验收紧
  - 明确只接受 `set / merge / append / delete`
  - 避免把不兼容动作混进批处理

### 3. 项目上下文与状态注入优化

- Agent 默认项目上下文现在会同时注入 `.project/AGENTS.md` 和 `.project/status/*.json`
- 缺少 `.project/AGENTS.md` 时，仍可回退到状态 JSON 作为默认真值层
- Workflow 执行链路和主 Agent 入口都接入了项目状态目录树读取能力
- 提示词中明确增强了 JSON 工具的推荐用法，优先鼓励模板补齐、历史追加和 patch 写法

### 4. 会话连续性与 Workflow Delta Memory 优化

- 会话历史构建增加任务记忆压缩能力
- 支持把目标、进展、约束、路径、已用工具等信息压缩成高密度摘要
- 支持模型参与生成更短的任务记忆摘要，并在失败时回退到规则摘要
- Workflow 执行提示改为注入裁剪后的 delta memory
- 上下文更聚焦“继续当前步骤最需要的信息”，减少冗余消息对后续步骤的干扰

### 5. 版本与发布更新

- 项目版本号升级到 `0.1.9`
- 已完成 Windows EXE 与 Android APK 打包
- 已生成 Windows updater 对应的签名和 `latest.json`
