# 神笔写作 v0.2.2 更新日志

发布日期：2026-04-25

## 版本亮点

本次更新围绕「创作台详情页收敛」「工作流详情编辑与运行链路拆分」和「Agent 会话执行层模块化」三条主线推进，同时补齐了创作台用量统计与发版版本同步。整体目标是降低详情页耦合度、提升运行链路可维护性，并让版本检查、桌面更新和双端发版产物保持一致。

## 重点更新

### 1. 创作台详情页拆分与批量梗概能力补齐

- 将扩写详情页的大块渲染逻辑拆分为更聚焦的明细片段组件
- 新增 `BatchOutlineVolumeDialog`，为分卷梗概批量生成提供独立交互入口
- 创作台详情相关 hooks 与元数据编解码逻辑已独立沉淀，便于后续继续扩展
- 页面加载态、错误态统一接入公共 `LoadingBlock` 与 `ErrorBlock`

### 2. 工作流详情编辑器与运行态代码重构

- 工作流详情页拆出 `detail`、`runtime`、`selectors`、`messages` 等独立模块
- 节点草稿、运行控制、移动端三栏切换与详情渲染的职责边界更清晰
- 工作流判断逻辑新增定向测试，降低后续修改分支控制时的回归风险
- 页面空态与异常态改为统一组件，提升可复用性与一致性

### 3. Agent 会话执行链路模块化

- 将 `session.ts` 中的异步控制、调试输出、子代理执行、工具集构建等逻辑拆分到独立文件
- `runAgentTurn` 只保留主回合编排职责，后续维护成本更低
- 子代理执行、工具请求、调试消息与部件拼装现在拥有更明确的模块边界
- 新增 `chatRunStore` 与相关运行时状态目录，为会话执行状态收口提供基础设施

### 4. 用量统计补齐创作台来源维度

- Rust 端新增 `record_expansion_usage` 命令，创作台请求可单独落日志
- 设置页用量统计支持按来源筛选：
  - 图书 Agent
  - 工作流
  - 创作台
- 统计卡片新增来源分组汇总，当前请求数与 Token 消耗更容易区分

### 5. 版本号与更新测试同步到 0.2.2

- 项目版本号由 `0.2.1` 升级到 `0.2.2`
- `package.json`、`package-lock.json`、`Cargo.toml`、`Cargo.lock`、`tauri.conf.json` 与 README 展示版本已同步更新
- 关于页、侧边栏、更新 store 与 App 中的版本断言已同步切换到：
  - 当前版本 `0.2.2`
  - 可用更新样例 `0.2.3`
