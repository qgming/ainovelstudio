# 神笔写作 v0.3.1 更新日志

发布日期：2026-05-23

## 版本亮点

本次更新是 `0.3.0 -> 0.3.1` 的一次功能迭代，主线是**为工作区引入文件关联（知识图谱）能力**：任意两个工作区文件之间可以建立带标签的无向多对多关联，既能在文件树上一眼看到引用关系，也能让 AI 在创作时自动获得相关设定文件作为上下文。

围绕这条主线，本版本同时包含：

- 后端新增关联表与级联维护，前端新增缓存层、Popover 编辑面板和编辑对话框
- 新增「关联图谱」页面，使用 ECharts 力引导布局可视化整本书的关联网络
- AI 提示词与工具链同步升级：可在 prompt 中自动注入关联文件，可调用工具直接维护关联
- 一批配套提示词、UI 间距、返回按钮目标的细节打磨

## 重点更新

### 1. 工作区文件关联：任意文件之间的无向多对多关系

- 后端新增 `book_workspace_relations` 表，支持 `(book_id, entry_a_path, entry_b_path, relationship)` 四元唯一约束
- 关联是**无向**的：内部强制 `entry_a_path <= entry_b_path` 规范化，避免重复
- 同一对文件可以有多条不同 `relationship` 标签的关联（如「出场人物」「引用设定」并存）
- 重命名 / 移动 / 删除文件时，关联表自动级联更新或清理（SQL CASE + LIKE 子树匹配，全部在同一事务）
- 规范化破坏时自动交换 a/b 两列，并清理因移动产生的自环关联

### 2. 文件树新增关联图标与 Popover 面板

- 文件树每一行（仅文件，非目录）在「更多」按钮左侧新增 `Link2` 图标
- 关联数 > 0 时图标常驻显示并带数量角标；为 0 时仅 hover 显示
- 点击图标弹出 Popover：展示该文件的所有关联（对端文件名 + 关系标签 + 编辑/删除）
- 点击对端文件名直接跳转到对应文件
- Popover 复用 `DropdownMenuContent`，与窗口边缘保留 12px 间距，移动端宽度自适应

### 3. 关联编辑对话框

- 新建 / 编辑统一对话框：目标文件下拉（沿用 `flattenTreeFiles` 过滤可编辑文本文件）+ 关系标签输入框 + 备注 textarea
- 编辑模式下目标文件锁定只读，仅可改标签与备注
- 备注支持「清空」语义：后端通过 `clearNote` 配套字段处理三态 Option，避免 JSON 序列化歧义

### 4. 关联图谱页面（新页面）

- 书架卡片右上角「更多」菜单新增**关联图谱**入口，路由 `/books/:bookId/relations`
- 使用 ECharts 力引导布局可视化整本书的关联网络（GraphChart + Canvas Renderer）
- 节点按文件夹首段（人物 / 势力 / 章节 / 设定等）自动分类着色，支持 legend 筛选
- 节点直径与度数成正比（5~30 范围），高连接度的核心设定一眼可见
- 边按起点 category 着色，hover 时展示关系标签和备注
- 力学参数参考 ECharts 官方「悲惨世界人物关系图」示例（repulsion=100, edgeLength=[40,120], gravity=0.02）
- 顶部使用统一的 `PageBackTitle`，返回按钮指向书架首页
- 支持暗色模式自动适配（监听 `html.dark` class 变化重渲）

### 5. AI 创作自动注入关联上下文

- `loadProjectContext` 新增 `readRelations` 回调，针对当前 active file 自动拉取关联清单
- 关联文件以**路径 + 关系标签**形式追加到上下文（path-only 策略，不展开内容），最多 12 条
- BOOK 模式规则中新增「项目入口」与「关联维护」两段，引导 AI 优先读取关联文件而不是盲目搜索
- turn context block 中关联文件额外标注「来源：作者已显式建立的关联」，AI 可据此判定置信度

### 6. AI 可直接维护关联（新工具集）

- 新增 `workspaceRelationToolset`，对外暴露 4 个工具：
  - `workspace_relation_list`：列出某个文件的所有关联
  - `workspace_relation_create`：新建两个文件之间的关联
  - `workspace_relation_update`：修改关系标签 / 备注
  - `workspace_relation_delete`：删除单条关联
- 工具内部直接调用前端 `bookWorkspaceApi` 包装，不重复实现 invoke 逻辑
- 写操作完成后通过 `onWorkspaceMutated` 触发文件树刷新，AI 创建的关联立即在 UI 上可见
- 项目模板的 `.project/AGENTS.md` 同步新增「文件关联」章节（仅影响新建的书）

### 7. 配套细节打磨

- Popover 与窗口边距修复：从「紧贴边框」改为留出 12px 间距
- 关联图谱页面切换图表库实现：从 `echarts-for-react` 切换到原生 `echarts.init` + `ResizeObserver`，解决 Vite + Tauri 环境下 `size-sensor` 失效导致页面空白的问题
- 关联图谱顶部布局与其他页面对齐（左侧返回按钮 + 标题）
- 返回按钮目标修正为书架首页 `/`，不再回到工作区

## 兼容与行为变化

- 数据库新增 `book_workspace_relations` 表，首次启动 0.3.1 会自动迁移，无需手工操作
- 项目模板新增的「文件关联」章节**只对新建书生效**，已有书的 `.project/AGENTS.md` 不会被自动覆盖
- AI 在 active file 上下文中会额外看到最多 12 条关联文件提示，可能影响 prompt token 消耗（沿用 path-only 策略，开销可控）
