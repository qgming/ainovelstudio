# 神笔写作 v0.4.0 更新日志

发布日期：2026-05-31

## 版本亮点

本次更新是 `0.3.1 -> 0.4.0` 的一次核心架构升级，主线是**把写作 Agent 内核迁移到 pi AgentHarness**。旧版自研 agent loop、AI SDK 工具封装、线性上下文拼装和伪 JSON 容错层被大幅收敛，新的运行时以 pi 的持久会话、工具协议、技能加载、事件流和压缩机制为中心。

这次换芯带来的直接收益：

- 长篇写作会话现在落到每本书自己的 pi jsonl 会话中，跨轮复用更稳定
- 压缩真正作用于模型读取的 pi 会话，而不是只压缩 UI 层聊天记录
- 工具调用改为 pi 原生 AgentTool + TypeBox schema，参数校验和失败语义更清晰
- 外部模型请求走 Rust 流式代理，绕开 WebView CORS 预检问题，保留 SSE 增量输出
- 系统提示词更稳定，每轮变化的运行时上下文改走 messages 层，利于前缀缓存命中

## 重点更新

### 1. pi AgentHarness 成为写作 Agent 主内核

- 引入 `@earendil-works/pi-agent-core` 与 `@earendil-works/pi-ai` `0.78.0`
- `runWritingAgentHarness` 接管写作回合执行，替代旧自研 `runWritingAgentPi` / `runAgentLoop`
- 每个应用 `sessionId` 对应一个确定性的 pi 持久会话，按书籍 `bookId` 归档到 `.sessions/`
- `AgentHarness` 负责 turn 队列、`followUp` 续轮、`waitForIdle`、工具调用钩子与会话事件
- App chat store 保留为 UI 渲染层，真实模型上下文以 pi session 为准

### 2. 长会话压缩改用 pi 原生 compact

- 自动压缩从旧的 ChatEntry 摘要改为 `harness.compact()`
- 手动压缩入口也临时 create-or-open 同一 pi 会话后调用 `compact()`
- 压缩结果直接写回 jsonl 会话，下一轮模型读取的就是压缩后的真实上下文
- 新增面向网文创作的压缩指令，保留主线目标、人物状态、世界观、风格、进展、未完成任务和关键文件路径
- 自动压缩只在整轮结束且 harness 回到 idle 后触发，失败时不阻断主流程

### 3. 工具系统迁移到 pi AgentTool

- 新增 `buildPiTools`，按当前启用工具组装 pi 原生工具列表
- 工作区工具通过 `createPiTool` 包装为 `AgentTool<TSchema>`
- 工具参数统一使用 TypeBox schema，支持 `prepareArguments` 预处理
- 工具失败语义调整为 throw，符合 pi 工具协议
- `ask_user` 改为专用 pi 工具，支持 awaiting / completed / failed 三态回传
- 新增 pi 工具桥接层，把内部 `ToolResult` 映射为 pi 的 `content/details`

### 4. pi 事件流适配到现有 UI

- 新增 `AgentEventAdapter`，负责把 pi `AgentEvent` 转成应用现有的 `AgentPart` 与会话事件
- 流式文本、思考内容、工具开始/更新/结束、usage 和错误状态都保持 UI 兼容
- assistant 消息 id 改用 pi `uuidv7`，降低连续续轮下的主键冲突风险
- 非 assistant 的 tool result / user 内部消息不再透出为额外助手气泡
- `ask_user` 工具中止或失败时会合成 failed 卡片，避免 UI 永久停在等待态

### 5. 模式控制改为统一 ModeConfig

- BOOK 与 autopilot 模式的续轮、步数预算和工具审批收敛到 `modes/*`
- autopilot 续轮通过 `harness.followUp()` 在同一次 prompt 流中继续推进
- 写作协议修复续轮新增硬上限，避免模型始终不发控制信号时无限补轮
- 高风险工具审批逻辑保留在模式层，runner 只负责调用策略

### 6. Rust 流式模型代理修复 CORS 与中断

- 外部 http(s) 模型请求改由前端 fetch 注入层转发到 Rust `stream_provider_request`
- Rust 侧用 reqwest 发起请求，并通过 Tauri Channel 把 SSE chunk 增量送回 WebView
- 只代理非本机的绝对 http(s) 模型端点，localhost、tauri.localhost、相对路径、blob/data/file 保持原生 fetch，避免启动白屏
- 支持 SDK abort / 读流 cancel 时调用 `cancel_provider_stream` 中断 Rust 流
- 保留 OpenAI SDK 的 ReadableStream 消费方式，国产中转网关即使没有 CORS 头也可正常流式输出

### 7. 提示词、技能与上下文进一步 pi 化

- 技能加载改用 pi 原生 `loadSkills`，从真实 skills 目录读取启用技能
- 系统提示词保持静态：人设、环境、工具、技能和模式信息固定在 system 前缀
- 日期、workspace、规划状态等每轮变化内容改由 `context` 钩子注入 messages 层
- 每次请求前清理旧的运行时上下文块，避免 messages 堆叠
- 项目内置写作技能和 AGENTS 指引同步调整，新增 `story-prose-craft` 技能

### 8. 书籍工作区与本地存储重构

- 书籍工作区后端拆出 `fs_store`、`index_schema`、`session_store` 等模块
- 书籍文件解析从路径根进一步收敛到稳定的 `bookId`
- 工作区内容搜索新增 grep 能力，便于 AI 和用户定位正文/设定片段
- 技能存储拆出 `skill_store`，与 pi 技能目录读取适配
- 移除调试设置页与 debug 后端域，减少发布版暴露面

## 兼容与行为变化

- 0.4.0 的 AI 会话上下文来源改为 pi jsonl 持久会话，旧 UI 聊天记录不再作为模型历史的唯一来源
- 压缩行为更接近真实模型上下文管理，长会话超窗时的表现会与 0.3.1 不同
- 外部模型请求会被 Rust 代理接管；本机开发服务、Tauri 同源资源和静态资源不会被代理
- 自研伪 JSON 解析器已移除，工具参数现在依赖 pi / TypeBox schema 校验
- Debug 设置页相关入口已移除
