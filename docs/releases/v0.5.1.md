# 神笔写作 v0.5.1 更新日志

发布日期：2026-06-07

## 版本亮点

本次更新是 `0.5.0 -> 0.5.1` 的一次稳定性修复版本，重点修复外部模型流式输出中 `: keepalive` 心跳行被误当成 JSON 解析的问题。

在部分模型网关的 SSE 响应中，服务端会插入形如 `: keepalive` 的注释/心跳行。这类内容是合法 SSE，但不是 JSON。旧版流式代理里存在针对特定分片行为的 SSE 缓冲与重组规则，可能让心跳行进入后续 JSON 解析链路，最终触发 `Unexpected token ':', ": keepalive" is not valid JSON`。

0.5.1 将 Rust 代理和前端注入层都改回原始字节透传，让 OpenAI SDK / pi-ai 按自身 SSE 协议解析流内容，避免本地代理层误处理网关心跳。

## 重点更新

### 1. 修复模型 SSE keepalive 解析错误

- 修复部分外部模型网关返回 `: keepalive` 心跳行时，前端报 `Unexpected token ':'` 的问题
- 移除本地流式代理里针对特定模型分片行为的特殊 SSE 缓冲与重组规则
- Rust `stream_provider_request` 现在直接把 `reqwest::bytes_stream()` 的原始字节块通过 Tauri Channel 回传
- 前端 `rustProviderFetch` 收到 chunk 后直接写入 `ReadableStream`，不再二次解码、拼接或按 SSE 消息边界拆分
- 保留原有 open / chunk / done / error 事件契约，以及 cancel / abort 中断能力

### 2. 更新版本与发布产物

- 项目版本元数据更新为 `0.5.1`
- Windows 安装包和 Android arm64 APK 已重新生成
- `release-packages/app.json` 已指向 `v0.5.1` 的 GitHub release 下载地址
- latest 包同步覆盖为当前版本，便于本地分发与上传

## 兼容与行为变化

- 外部模型流式代理不再尝试修复或重组 SSE 消息边界，后续解析完全交给上游 SDK
- `: keepalive`、空事件、注释行等合法 SSE 内容不再由本地代理层特殊处理
- 对正常模型输出、HTTP 非 2xx 错误 body、读流中断和用户取消的处理语义保持不变
- 本次更新不涉及书籍项目结构迁移，不应改变已有正文、设定或 Markdown 项目资料
