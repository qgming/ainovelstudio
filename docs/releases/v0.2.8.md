# 神笔写作 v0.2.8 更新日志

发布日期：2026-05-14

## 版本亮点

本次更新从 `0.2.7` 推进到 `0.2.8`，核心目标是把代理系统从“能跑”继续收敛到“更稳、更清楚、更像一个真正可协作的写作搭档”。这一轮同时完成了写入工具链升级、工具命名统一、消息结构扩展、项目状态维护简化、提示词系统瘦身，以及设置页 AGENTS 的一键回退能力。

## 重点更新

### 1. 写作代理的执行链路更稳了

- 新增面向工作区的创建能力，补齐文件创建与正文写回链路。
- 写入工具链重构后，代理在创建、追加、覆盖、局部编辑之间的选择更清楚，恢复执行时也更稳定。
- 工具调用恢复机制得到加强，异常中断后更容易继续当前任务，而不是把上下文重新打散。
- 配置验证和错误处理进一步强化，模型配置异常、工具参数异常和运行时错误都会更早暴露，并以更清楚的方式反馈到界面。

### 2. 工具命名、工作区 API 和状态维护策略统一收口

- 内置工具命名统一到现在这套工作区语义，例如读取工作区文件、搜索工作区、写入文本、路径操作、JSON 数据等，减少旧命名和新命名并存带来的歧义。
- 工作区操作 API 整体重构，读、写、路径、JSON、字数统计等能力边界更清楚。
- 长篇工作区模板和状态维护策略做了简化，不再默认铺开过重的目录结构，项目初始化更轻，后续维护也更容易。
- 项目默认上下文、状态文件与工作流规则的配合更一致，代理在读取 `.project/AGENTS.md`、`.project/README.md` 与状态 JSON 时更可预期。

### 3. 消息结构升级，代理能表达更完整的过程

- 消息内容结构升级为多部分内容模型，支持把正文、工具结果、结构化片段和独立推理分开承载。
- 独立推理片段现在可以和普通文本更自然地共存，后续无论是 UI 呈现还是上下文压缩都更容易做细化处理。
- 这次升级也为后续更细颗粒度的消息渲染、上下文拼装和代理协作打下了基础。

### 4. 提示词系统整体重写，主代理更像 Codex 风格的协作者

- 内置主代理 `AGENTS.md` 重新整理为“身份 / 个性 / 使命 / 工作原则 / 工作循环 / 工具边界 / 写作规则”的结构，减少重复规则堆叠。
- system prompt 从厚重说明书收敛成更短更硬的 Agent OS 内核，只保留共享工作区、执行循环和运行边界。
- 当前轮动态上下文瘦身为更接近“状态包”的形式，重点保留本轮事实、任务类型、输出预期和最小提醒。
- 最终传给模型的提示词结构统一为 Markdown 标题格式，不再暴露 `s00`、`s01` 这类内部 section key。
- 动态资源目录与技能说明也一并压缩，减少和主代理 AGENTS 的重复表述。

### 5. 设置页现在可以一键恢复内置 AGENTS

- 设置页的 `AGENTS.md` 编辑区在保存按钮左侧新增“重置”按钮。
- 点击后会直接把当前主代理 AGENTS 恢复为内置版本，并立即保存，而不是只回填编辑框草稿。
- Rust 端新增 `reset_default_agent_config` 命令，确保恢复来源是打包内置模板本身，而不是前端 fallback 文本。

### 6. 字数统计与界面反馈进一步补强

- 新增小说字数统计能力，便于代理在创作、审稿和状态维护时做更直接的量化判断。
- Agent 面板、上下文概览和错误提示区域都补了更清楚的反馈，运行状态和失败原因更容易看懂。

### 7. 版本与文档同步到 0.2.8

- 项目版本已同步更新到 `0.2.8`。
- 同步更新 `package.json`、`package-lock.json`、`Cargo.toml`、`Cargo.lock` 和 `tauri.conf.json`。
- README 的版本徽章、下载链接和更新日志入口已同步到 `0.2.8`。
- 更新检测相关测试已调整为以 `0.2.8 -> 0.2.9` 作为“发现新版本”的验证场景。

## 影响范围

本次更新覆盖以下几条主线：

- 代理提示词与协作风格
- 工具定义与工作区 API
- 消息内容结构与上下文拼装
- 长篇工作区模板与状态维护策略
- 设置页 AGENTS 配置体验
- 版本同步与更新检测测试
