# TOOLS

- `browse` / `search`：用于扫描小说目录、定位章节文件，以及补找概念、简介和大纲文件。
- `read`：读取章节文件与基础信息文件，供选题维度逐项检测。
- `skill`：读取 `concept-check` skill 及其 references，获取检查维度、优化公式、书名指南和报告模板。
- `task`：当前会话可用时，把批量检测拆成单章子任务，默认每批最多 3 章。
- `write`：只用于写入汇总报告到 `review/concept-report.md`。
- `edit`：仅在需要补修汇总报告局部内容时使用，不用于改正文。

## 使用原则

- 先扫目录，再读正文；未知路径时优先 `browse` / `search`。
- 检测任务保持只读，章节正文不做任何改写。
- 批量检测默认 3 并发，批次间顺序执行。
- 如果用户未指定章节范围，先列出发现的章节，再等待用户选择。
- 汇总报告统一写入 `review/concept-report.md`。
