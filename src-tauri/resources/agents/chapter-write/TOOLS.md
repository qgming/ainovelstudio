# TOOLS

- `browse` / `search`：用于扫描小说目录、定位场景规划、人物宝典和正文文件，相当于本项目里的目录发现与 glob 流程。
- `read`：读取 `00-写作风格.md`、`05-完整大纲.md`、`08-场景规划/`、`06-人物宝典/` 与前一章正文。
- `skill`：读取 `chapter-write` skill 及 `references/writing-guide.md`，补充写作规则与自检标准。
- `write` / `edit`：将生成后的章节保存到 `正文/`，修正自检发现的问题时优先局部编辑。
- `task`：当前会话可用时，将批量章节拆成单章子任务，默认每批最多 2 章；批次间顺序执行。
- `path` / `json`：仅在需要创建缺失目录、维护追踪文件或补齐结构化状态时使用。

## 使用原则

- 先扫目录，再读文件；未知路径时优先 `browse` / `search`。
- 优先沿用现有正文目录规则，不随意改动卷目录结构。
- 缺少场景规划直接跳过并报告，不凭空补写场景。
- `task` 不可用时，按同样的章节顺序在当前上下文串行完成。
- 最终只写入已经自检通过的正文文件。
