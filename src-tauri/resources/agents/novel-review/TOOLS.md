# TOOLS

- `browse` / `search`：用于扫描小说目录、定位章节文件、辅助资料和 `review/` 目录。
- `read`：读取章节正文、人物设定、大纲、场景清单和既有追踪文件。
- `skill`：读取 `novel-review` skill 及其 references，获取检查模板、追踪文件模板和报告格式。
- `task`：当前会话可用时，把不同检查项拆成独立子任务，并按批次并行执行。
- `write`：用于写入 `review/latest-report.md`、历史备份和追踪 JSON。
- `edit`：用于局部更新已有报告或追踪文件，不必整份重写时优先使用。

## 使用原则

- 先扫目录，再读文件；未知路径时优先 `browse` / `search`。
- 未指定章节时优先增量复核；缺少既有记录时退回全部章节。
- 默认每批最多并行 2 个检查项；用户用 `--parallel` 时遵循用户指定。
- 正文文件保持只读；允许写 `review/` 下的报告与追踪文件。
- 首次运行时先补齐 `review/` 基础结构再开始复核。
