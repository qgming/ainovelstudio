# 题材调研拆文

面向选题前期的轻量调研工作流。一次循环跑完：扫榜 → 拆爆款 → 输出选题简报，让用户拿着简报直接进入立项阶段。

## 适用场景

- 想换赛道但不确定写什么
- 看到市场风向变化想验证一下能不能跟
- 想给某个题材方向找几本对标爆款
- 立项前想要一份系统化的选题依据

## 默认流程

1. **市场扫描**（market-scout）按主提示词扫平台风向，给出推荐方向 + 对标书候选
2. **对标拆文**（story-analyst）挑 1-2 本最值得追的对标书做结构拆解
3. **选题简报**（story-analyst）整合扫描与拆解，输出可立项的选题简报
4. 完成 → 结束（不循环）

## 设计要点

- 默认 maxLoops = 1，调研一轮即收口，避免无限拆下去
- 调研结果统一写入 `.project/MEMORY/market/` 与 `.project/MEMORY/analyses/`
- 不动正文，纯产出供作者决策的资料
- 拆爆款上限 1-2 本，控制时长，避免变成系统拆书工程

## 节点侧重

- `scan-market`：本轮市场风向 + 推荐方向 + 对标书候选
- `analyze-benchmark`：对标书结构 / 节奏 / 爽点 / 反转拆解
- `produce-brief`：可立项的选题简报

## 建议用法

- 主提示词里写清目标体裁（长篇 / 短篇）、目标平台、自身擅长与禁区
- 跑完后用 `long-novelist` 或 `short-novelist` 进入立项阶段
- 后续如果方向变化，再跑一轮即可
