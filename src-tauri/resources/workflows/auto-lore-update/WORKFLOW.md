# 全自动设定更新

这是一个专门用于维护人物、背景、时间线和追踪资料的内置工作流模板。

## 适用场景

- 章节已经持续产出，需要定期整理设定
- 想把人物档案、背景说明、时间线和项目追踪集中更新
- 希望每轮更新后都经过一次结构化审查

## 默认流程

1. 设定整理代理扫描最新章节与资料并执行更新
2. 设定审查代理检查更新结果是否完整且一致
3. 审查失败则回到更新步骤重新整理
4. 通过后由循环控制决定是否继续下一轮

## 绑定建议

- 推荐绑定已有多章正文和基础设定文件的书籍工作区
- 首次运行前检查角色设定、背景资料目录是否已经存在
- 如果只想做单轮整理，可以把最大循环次数调低

## 输出要求

- 审查步骤必须返回结构化 review_json
- 工作流模板本身不保存运行历史
- 物化后的 runtime workflow 可按项目需要继续编辑
