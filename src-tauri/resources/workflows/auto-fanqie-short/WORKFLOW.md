# 全自动番茄短篇

这是一个面向番茄平台 1-3 万字短篇的全自动批量生产模板。它不是“给定范文后生成一本”，而是“每轮循环自动随机立项一本新短篇”，再把立项、写作、终审串成一条线。

## 适用场景

- 想无人值守批量生产番茄平台短篇
- 想每轮自动生成一个全新的随机题材短篇
- 想让工作流自动完成立项、开篇、续写、终审
- 想把每本短篇都沉淀成独立项目目录，方便后续筛选和发布

## 绑定前准备

把绑定工作区当作“短篇工厂根目录”即可。

不需要预先放：

- 范文
- 创作方向
- 题材设定
- 书名
- 人设

这些内容都由工作流在每轮开始时自动随机生成。

## 默认流程

1. `fanqie-style-genome` 自动新建立项并锁定本轮随机题材
2. `fanqie-ip-architecture` 生成阶段二原创架构与台账
3. `fanqie-opening-three` 写出前 3 章与开篇自检
4. `fanqie-short-batch` 自动写完剩余章节并整合全本
5. `fanqie-final-audit` 输出发布终稿并做最终决策
6. 终审未通过时，工作流回到阶段四返工当前项目；通过后自动开始下一轮新短篇

## 固定产物

每轮都会新建一个独立项目目录，建议结构类似：

- `短篇工厂/短篇-0001-书名/`
- `短篇工厂/短篇-0002-书名/`

每个项目目录中都会产出阶段文件、正文和终稿。

## 运行语义

- 所有节点都先读文件，再写文件。
- 绑定工作区是工厂根目录，不是单本书目录。
- 每轮开始都要新建一个项目目录。
- 本轮随机生成主赛道、题材、背景、关系、卖点、字数和章节数。
- 阶段文件是当前轮项目的长期事实源，后续节点默认信任上游固定产物。
- 阶段五会写文件，也会通过 `workflow_decision` 决定是否返工。
- 返工默认回到阶段四，只处理当前轮项目。
- 当前轮通过终审后，工作流继续下一轮，自动开始下一本新短篇。
