# 全自动番茄短篇

这是一个面向番茄男女频短篇的内置工作流模板，把“拆范文 → 做原创架构 → 写黄金开篇 → 批量写完全本 → 发布前终审”串成一条线。

## 适用场景

- 已经准备好番茄爆款范文，想做 1:1 风格建模
- 想把短篇创作拆成阶段文件，降低 AI 漂移
- 想一次性跑完整本短篇，不逐章手动催更
- 想在发布前同步拿到 AI 风险、合规、签约和运营建议

## 绑定前准备

建议在工作区预先放好：

- `00-输入/01-番茄范文.md`
- `00-输入/02-创作方向.md`

创作方向里建议写清：

- 主赛道（女频 / 男频 / 中性短剧向）
- 时代背景
- 核心关系类型（恋爱 / 宿敌 / 搭档 / 群像 / 事业线）
- 核心卖点

## 默认流程

1. `fanqie-style-genome` 生成阶段一文风图谱
2. `fanqie-ip-architecture` 生成阶段二原创架构与台账
3. `fanqie-opening-three` 写出前 3 章与开篇自检
4. `fanqie-short-batch` 自动写完剩余章节并整合全本
5. `fanqie-final-audit` 输出发布终稿并做最终决策
6. 终审未通过时，工作流回到阶段四返工；通过后结束

## 固定产物

- `10-阶段一/01-文风基因图谱报告.md`
- `20-阶段二/01-IP架构与工程化台账.md`
- `30-阶段三/01-黄金开篇三章.md`
- `30-阶段三/02-开篇自检与台账更新.md`
- `40-阶段四/01-全本生成执行计划表.md`
- `40-阶段四/02-逐章自检与台账.md`
- `40-阶段四/03-全本终稿.md`
- `50-阶段五/01-发布终稿.md`
- `50-阶段五/02-终审与运营建议.md`

## 运行语义

- 所有节点都先读文件，再写文件。
- 阶段文件是长期事实源，后续节点默认信任上游固定产物。
- 执行前先锁定主赛道与主叙述者类型，后续阶段统一沿用。
- 阶段五会写文件，也会通过 `workflow_decision` 决定是否返工。
- 返工默认回到阶段四，由批量生成代理处理受影响章节和终稿。
