# 全自动番茄短篇

这是一个面向番茄平台 1-3 万字短篇的全自动批量生产模板，已经切换到当前新的内置编辑链。它不是依赖专用旧代理的五阶段模板，而是由通用编辑角色完成随机立项、建档、搭纲、成稿、终审和发布整理。

## 适用场景

- 想无人值守批量生产番茄平台短篇
- 想每轮自动生成一个全新的随机题材短篇
- 想让工作流自动完成立项、建档、搭纲、正文、终审和发布整理
- 想把每本短篇沉淀成独立项目目录，方便筛选、返修和发布

## 绑定前准备

把绑定工作区当作“短篇工厂根目录”即可。

不需要预先放：

- 范文
- 创作方向
- 题材设定
- 书名
- 人设

这些内容都由工作流在每轮开始时自动生成和维护。

## 默认流程

1. `chief-editor` 自动立项并创建新的短篇项目目录
2. `lore-editor` 为该项目建立最小可用的设定底盘和状态文件
3. `outline-editor` 生成适配番茄短篇的全本大纲与章节拆分
4. `chapter-editor` 拆清开篇钩子、前 3 章节拍和后续写作节奏
5. `serial-writer` 完成整本短篇成稿
6. `review-editor` 输出终审结论，并通过 `workflow_decision` 决定是否返工
7. 未通过时，工作流回到 `serial-writer` 返修当前项目
8. 通过后，`polish-editor` 整理最终发布稿、摘要和项目状态
9. 结束节点根据最大轮次决定是否自动开始下一本

## 固定产物

每轮都会新建一个独立项目目录，建议结构类似：

- `短篇工厂/短篇-0001-书名/`
- `短篇工厂/短篇-0002-书名/`

每个项目目录中都会产出立项文件、设定文件、大纲文件、章节规划、正文和发布稿。

## 运行语义

- 所有节点都先读文件，再写文件
- 绑定工作区是工厂根目录，不是单本书目录
- 每轮开始都要新建一个项目目录，并尽量避开最近几轮重复题材
- 本轮会自动生成主赛道、题材、背景、关系、卖点、字数和章节数
- 立项、设定、大纲和章节规划是当前轮项目的长期事实源
- 阶段末的终审会通过 `workflow_decision` 决定是否返工
- 返工默认回到正文成稿节点，只处理当前轮项目
- 当前轮通过终审并完成发布整理后，工作流继续下一轮，自动开始下一本新短篇

## 节点侧重点

- `project-setup` 节点负责随机立项、避重和项目目录初始化
- `build-canon` 节点负责最小设定底盘与状态层
- `build-outline` 节点负责短篇结构、反转和 payoff
- `opening-plan` 节点负责开篇三章和章末驱动力
- `draft-short` 节点负责整本成稿与返修
- `final-review` 节点负责可发布性判断
- `polish-release` 节点负责发布整理、终稿和工厂索引收口

## 其他说明

- 所有节点都不要求严格输出 JSON
- 工作流本身不自动绑定工作区，进入详情页后由用户手动绑定
- 左侧基础区可直接设置最大循环次数，并支持切换为无限
- 运行历史和步骤复盘保存在数据库中，不写回模板文件
