# story-deslop

把 AI 味浓重的文本改回人味，最少改动达到最大效果。

## Use When

- 用户要清 AI 味 / 终稿润色。
- 终稿编辑步骤的润色任务。
- 长篇连载或短篇集的终稿整理任务。

## Inputs To Read

- 待润色的文件全文（用 `workspace_read`）。
- `.project/README.md` 与 `.project/memory/` 的风格基线（用户文风、字数约束、禁写约束）。

## Procedure

### 三遍法

1. **第一遍：识别空泛大词与宣传腔**
   - 标出"基本""略有""稍微""相对而言""总体来说"等无信息词。
   - 标出"展现""彰显""诠释""映衬""弥漫"等书面腔。
   - 用 `workspace_edit` 做最小替换：能换词不换句。

2. **第二遍：拆三连句与机械排比**
   - 整齐对仗的三连句拆成参差的 1-3 句。
   - 删冗余形容词；动词带细节比形容词带情绪好。

3. **第三遍：还原口语跳跃感与节奏**
   - 段落长短交替，1 句独占段允许；4-6 句一段是 AI 味重灾区。
   - 查句长方差：一段里每句字数都在 ±5 字内、读着匀速平铺，就是 AI 节奏。砍出 1-3 字短句、保留一个长句，制造跳跃。
   - 60% 对话不带"说道 / 问道"，用动作替代标签。

### 守则

- 只改"怎么说"，不改"说什么"。剧情 / 人设 / 走向一概不动。
- 严守作者文风：作者口语化的不要改成书面，反之亦然。
- 改最少的字，让整段文字的味变过来。

## Detection Gates

### Gate A：禁用词与高频套话

- 查 `眼中闪过一丝`、`嘴角勾起`、`深吸一口气`、`不禁`、`仿佛`、`弥漫` 等高频 AI 味词。
- 处理方式优先级：删除无信息词 → 换成具体动作 → 保留原句但打碎节奏。

### Gate B：句式套路

- 检查 `……，带着……`、`他知道……`、`这一刻……`、连续排比、连续总结句。
- 不做同义词替换游戏；要改句子结构和叙事重心。

### Gate C：心理外化

- 把“他很紧张 / 愤怒 / 失落”换成动作、停顿、视线、手部反应、选择。
- 情绪强时可以短句独立成段，但不要解释“为什么这很重要”。

### Gate D：对话人味

- 删除多余“说道 / 问道 / 淡淡道”。
- 角色可以打断、答非所问、留半句；保留信息差。

### Gate E：结尾收束

- 删总结、升华、点题式收尾。
- 用动作、危险、选择、未说完的话收尾。

### Gate F：节奏指纹（统计层面的 AI 味，最难察觉也最致命）

换词治标，节奏指纹才是检测工具真正在看的（句长方差、突发性、均匀度）。词换干净了节奏还匀速，照样一眼 AI。

- **句长方差**：抽一段，看每句字数是不是都差不多。都在 ±5 字内 → AI 节奏。砍出 1-3 字短句、保留一个长句，打出跳跃感。
- **段落均匀度**：连续多段都是工整的 4-6 句 → 重灾区。插入独句成段、对话独占段，打碎工整。
- **突发性**：同一个词或句式在很近的距离里重复冒出 → 换掉或删掉其一。
- **爱总结 / 抒情**（网文 AI 味头号问题）：动作或对话之后，是不是兀自冒出一句归纳、升华、内心独白、展望未来（如打斗后突然总结"这一刻他懂了信赖的意义"）→ 删，用动作 / 对话 / 留白收，让读者自己悟。
- **过度完美也是破绽**：零口语、零断句、全程平稳 = AI。人味是刻意的不均匀——口语混入、偶尔断句、答非所问、半句留白。

详见 `references/rhythm-fingerprint.md`。

## Report Format

润色节点的最终回复只保留高密度摘要：

- 修改文件路径。
- 修改类型：禁用词 / 句式 / 心理外化 / 对话 / 结尾。
- 代表性改动 2-4 条。
- 字数变化与残余风险。

## Outputs / Write-Back

- 当前文件（用 `workspace_edit` 做最小修改）。
- 完成后用 `text_stats` 核对字数变化在合理范围。

不要写：新章节、设定文件、项目记忆。
