# 连续性体检清单

逐项核对。每发现一处冲突，记录：类型 / 位置（章节+锚点）/ 冲突双方事实 / 严重度 / 最小修订建议。

工具用法：精确查某个名字 / 术语 / 能力的**全部出现**用 `workspace_grep`（字面量或正则）；语义召回某条线相关章节用 `workspace_search`。

---

## 一、设定冲突

检查同一设定在不同章节是否自相矛盾。

- **能力 / 规则**：金手指、功法、体系规则前后是否一致？有没有「这章能做、那章又不能」？
- **地理**：地点的方位、距离、归属是否前后一致？
- **组织 / 势力**：名称、层级、人员归属、敌友关系有没有漂移？
- **物品 / 道具**：功能、来源、数量、下落是否对得上？
- **名称一致性**：人名 / 地名 / 术语有无错别字、异写、中途改名（`workspace_grep` 查全部出现最有效）。

> 排查锚点：用 grep 列出某术语全部出现 → 逐条比对定义是否一致。

## 二、伏笔失收

对照伏笔台账（`type: foreshadow`）核对。

- **埋了没收**：台账「待回收」里有没有早该回收却拖着的？预计回收章是否已过？
- **收了没埋**：某个回收 / 反转，前文有没有至少 1-2 处铺垫？凭空翻盘是硬伤。
- **回收突兀**：铺垫和回收之间断层太大，读者接不上。
- **台账漏登**：正文里埋了新伏笔但台账没登记，或回收了但台账状态没更新。

> 输出时一并提示更新台账：待回收→已回收、登记新伏笔及预计回收章。

## 三、人设崩坏

- **性格一致**：人物性格、价值观、说话方式前后是否稳定？（参考 character 记忆）
- **动机合理**：当前行为是否符合已立动机？有没有「为了剧情需要」强行 OOC？
- **能力 / 身份**：人物的身份、关系、已知信息是否和前文一致？会不会突然知道没读过的秘密？
- **声音区分**：多个角色台词是否还保持各自的辨识度，有没有趋同成一个腔调？
- **主角不塌**：主角有没有长期软弱、空转、只会懵（违背网文主角立场）？

## 四、时间线错乱

- **事件先后**：A 发生在 B 之前的设定，后文有没有写反？
- **时长 / 季节**：经过的天数、季节、节气是否自洽？「三天后」却描述了一周的事？
- **年龄 / 代际**：人物年龄、辈分、入门时间是否对得上？
- **并行线对齐**：双线 / 多线叙事时，各线时间节点能不能对齐？

## 五、能力越级

- **主角成长曲线**：实力提升是否有过程和代价，还是跳跃式越级？
- **战力守恒**：之前打不过的，现在能打过，中间有没有交代变化？反之亦然。
- **敌强弱序**：反派 / 对手的强弱是否随剧情合理排序，有没有「前期 BOSS 不如后期杂兵」的塌方？
- **边界**：角色有没有用出超过其设定边界的能力？

---

## 严重度分级

- **硬伤**：读者会直接出戏 / 弃书（设定自相矛盾、人设大崩、伏笔凭空回收、战力塌方）。优先修。
- **中瑕**：细心读者会发现（时间小错、名称异写、铺垫略薄）。择机修。
- **小疵**：基本无感（措辞微调）。可不动或顺手修。

## 修订原则

- 优先改**晚出现**的那一处，或改台账 / 记忆，让成本最低、影响最小。
- 不为修一处而引发新的连续性问题（改完顺手再查一遍关联点）。
- 作者**有意**的误导 / 反转不是 bug——拿不准时标「疑似有意，请作者确认」，不擅自当错误改掉。
