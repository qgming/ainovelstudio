# 番茄短篇 IP 架构

用于阶段二。目标是把阶段一的风格结论转成可写作、可追踪、可闭环的原创故事骨架。

## 必读输入

- `10-阶段一/01-文风基因图谱报告.md`
- `00-输入/02-创作方向.md`

## 固定输出

- `20-阶段二/01-IP架构与工程化台账.md`

## 执行原则

1. 全程锁定阶段一确认的第一人称主角视角。
2. 人设标签控制在 2-3 个，每个标签必须绑定可验证行为。
3. 主关系线 / 主冲突线、名场面、金手指全部绑定人物职业、经历、缺陷或怪癖。
4. 6 张台账字段必须完整，后续章节可逐章更新。
5. 结尾必须附【100%原创性自检报告】。

## 输出重点

- 主角与核心对位角色 400 字级全维人设卡
- 关系张力 / 阵营张力 / 宿命羁绊
- 与目标篇幅匹配的主关系线或主冲突线发展模型
- 主线冲突、情感冲突、视觉化名场面
- 只允许 1 个金手指，并写清使用规则与限制
- 6 张工程化台账的空表模板

## 赛道分支规则

先在文档开头明确：

- 主赛道：`女频` / `男频` / `中性短剧向`
- 主结构：`恋爱双核` / `主角+宿敌` / `主角+搭档` / `主角+群像`

### 角色输出选择

- 女频：默认写 `主角（叙述者）` + `感情对象`
- 男频：默认写 `主角（叙述者）` + `核心对位角色`
  - 核心对位角色可为：宿敌 / 女主 / 搭档 / 师父 / 反派
- 中性短剧向：按冲突密度最高的组合选择第二核心角色

## 输出合同

### 输出结构

1. 主角原创人设构建
2. 核心对位角色构建
3. 关系 / 阵营张力设计
4. 主关系线或主冲突线发展模型
5. 核心冲突设计
6. 金手指 / 专属设定
7. 6 张工程化台账
8. 5 个候选书名
9. 【100%原创性自检报告】

### 主角必须包含

- 基础信息
- 2-3 个核心标签 + 每项 3 个行为
- 核心共情点
- 成长弧光
- 底线与行为红线
- 说话习惯
- 潜台词预设
- 一句话人设钩子

### 核心对位角色必须包含

- 基础信息
- 2-3 个核心标签 + 每项 3 个行为
- 核心亮点维度
- 行为红线
- 说话习惯
- 潜台词预设
- 一句话钩子宣言

### 主关系线 / 主冲突线模型

按阶段一锁定篇幅选择模型，并为每个阶段写清：

- 章节区间
- 关系或冲突递进
- 情绪核心
- 2 个浅层情绪点 / 爽点 / 危机点
- 1 个深层名场面
- 阶段结尾钩子

### 工程化台账字段

表1 主角状态卡：

- 章节序号
- 主角物理状态
- 主角心理状态
- 当前核心诉求
- 说话习惯一致性校验结果
- 人设红线触碰情况

表2 核心对位角色状态卡：

- 章节序号
- 核心对位角色物理状态
- 对主角的态度变化
- 当前核心动作
- 说话习惯一致性校验结果
- 人设红线触碰情况

表3 伏笔索引表：

- 伏笔编号
- 埋笔章节
- 伏笔内容
- 预计回收章节
- 回收状态
- 跨章未提次数

表4 关系 / 阵营变化记录表：

- 章节序号
- 关系距离变化
- 信任度
- 情绪核心
- 糖点 / 爽点 / 压迫点专属细节
- 情绪递进校验结果

表5 AI 特征违规检查表：

- 章节序号
- 检查项
- 是否违规
- 违规内容
- 修改方案
- 违规趋势统计

表6 番茄合规检查表：

- 章节序号
- 合规检查项
- 是否违规
- 违规内容
- 修改方案

### 书名要求

- 生成 5 个
- 每个 12-18 字
- 包含 1-2 个流量关键词
- 自带情绪词或悬念词

## 质量门槛

- 所有设定具体、可验证、能直接拿去写章
- 不使用空泛标签堆砌
- 台账字段完整，可追溯
- 原创性自检覆盖背景、人设、情节、主题、情绪点、侵权风险
