# story-long-analyze

拆透别人的爆款，提炼可直接拿来用的写作模板。

## Use When

- 用户提到具体长篇并要求拆解、对标分析、研究黄金三章 / 节奏。
- 调研工作流的对标拆文与选题简报节点。

## Inputs To Read

- 对标书的公开正文 / 简介 / 平台数据：用 `web_search` 拿链接，再用 `web_fetch` 读章节。
- 本轮 `scan-*.md`（若来自调研工作流）：拿到对标书候选与方向。
- 已有 `analyses/` 拆解：避免重复拆同一本。

## Procedure

### 快速拆解（默认）

1. 拿黄金三章原文，分析钩子 / 期待感 / 信息释放 / 章末追读力。
2. 拿全书前 1/4 章节梗概，画情绪曲线与节奏模型。
3. 整理对标书档案、可借鉴模块、不要照抄部分。
4. 写入 `.project/MEMORY/analyses/{对标书}.md`。

### 深度拆解（用户要求逐章）

1. 按卷或按 30 章为一段拆解，每段产出节奏曲线 + 关键转折 + 爽点节奏。
2. 输出结构化条目，引用具体章节号。

### 选题简报（在调研工作流的简报节点）

1. 读本轮 scan + analyses，整合成可立项简报。
2. 写入 `.project/MEMORY/market/brief-*.md`：推荐题材 / 平台 / 读者 / 卖点 / 对标书 / 借鉴技法 / 差异化 / 风险与禁区。

## Breakdown Checklist

- **黄金三章**：开头钩子、主角第一印象、目标建立、爽点兑现、章尾追读力。
- **人物架构**：主角目标、对手压力、配角功能、关系变化。
- **爽点结构**：铺垫、压迫、误导、释放、后果。
- **节奏地图**：每卷核心冲突、升级节点、低谷和高潮。
- **可借鉴模块**：只抽象成结构方法，不复制桥段、人设、设定名。
- **风险提示**：哪些模块依赖作者文笔、题材知识或平台红利，普通项目不能硬搬。

## Output Requirements

- 拆解报告必须引用具体章节 / 场景 / 链接。
- 每个结论要给证据，证据不足标「推测」。
- 简报必须能直接指导下一步开书：题材、卖点、读者、差异化、风险。

## Outputs / Write-Back

- `.project/MEMORY/analyses/{对标书}.md`
- `.project/MEMORY/market/brief-*.md`（在简报节点）

不要写：正文、大纲、设定。

## Evidence Rules

- 拆解必须基于公开正文或可验证二手资料；不要凭印象编节奏曲线。
- 引用具体章节或场景时标章节号 / 位置。
- 区分「可借鉴」与「不要照抄」。
