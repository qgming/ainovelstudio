# continuity-check

## 目标

- 在章节完成后，或大纲与设定更新后，对正文和结构做可执行的质量检测。
- 让检查建立在现有正文事实、canon、大纲、章节目标和最新变更之上。
- 找出违背设定、规范、因果链和伏笔安排的问题。
- 找出场景前后状态不一致、场景后果丢失和场景信息维护缺口。
- 输出简洁、可复用、可直接返修的检查结果。

## 优先使用时机

以下情况优先调用本 skill：

- 一个章节已经写完，准备检查是否能进入修订或下一章
- `outline-manager` 更新了全书纲、卷纲、角色弧线或伏笔安排
- `story-bible` 更新了人物、世界、时间线、术语或其他 canon
- 用户担心章节违背人设、世界规则、时间线或大纲
- 用户想检查这一章的情节推进、场景衔接或信息释放是否合理
- 用户刚修改了关键场景、地点状态或场景内已暴露信息，想确认有没有连锁影响
- 用户想确认伏笔有没有埋偏、漏回收或回收过早
- 用户希望按固定规范审查章节完成度

如果任务重点是修正文案本身，交给 `story-writer`；如果重点是重整全局大纲，交给 `outline-manager`。

## 工作流程

1. 阅读用户需求，以及本次已完成的章节正文、最新大纲、最新设定和相关目标。
2. 判断本次检查涉及哪些维度：人物、情节、连贯性、场景状态、伏笔。
3. 必须读取对应的 `references/` 检查模板，再开始审查。
4. 提取当前已成事实的内容：人物状态、时间顺序、空间位置、场景状态、已发生事件、已释放信息、已出现线索和刚更新的规则。
5. 对照 `story-bible` canon、`outline-manager` 大纲、`chapter-planner` 章节目标和用户给定规范。
6. 检查新变更是否与既有正文、既有大纲、既有设定和已维护的关键场景打架。
7. 检查场景变化是否被持续维护：破坏是否延续、道具是否还在、人物知道的信息是否一致、地点限制是否被记住。
8. 标出冲突、歧义、漏项和高风险点，并区分严重程度。
9. 输出问题清单、证据来源、影响范围和建议修法。

## 强制读取规则

在使用本 skill 时，必须先读取与任务相关的参考资料：

- 检查人物动机、行为、称呼、已知信息、角色边界和配角功能：读取 `references/characters.md`
- 检查情节推进、因果链、目标达成、题材匹配、辨识度和世界观呈现：读取 `references/plot.md`
- 检查时间线、场景衔接、信息前后文和物理连续性：读取 `references/continuity.md`
- 检查伏笔、回收、暗示、公平性和悬念兑现：读取 `references/foreshadowing.md`

如果任务涉及关键场景被改动、反复回访、破坏、重建、封锁、掉马或信息暴露，优先补查：

- `story-bible` 中相关地点或关键场景记录
- `chapter-planner` 中相关场景清单与关键场景详细规划

如果题材属于以下类型，必须额外读取对应专项清单：

- 系统流 / 数值流：读取 `references/system-novel.md`
- 无限流 / 副本流：读取 `references/infinite-flow.md`
- 仙侠 / 修炼流：读取 `references/xianxia.md`
- 悬疑 / 推理：读取 `references/mystery.md`

如果任务范围不明确，至少先读取 `references/continuity.md`，再补读最可能相关的其他模板。

## 需要检查的输入

- 用户说明
- 已完成章节的正文
- `story-bible` 中的人物、世界、时间线和术语 canon
- `story-bible` 中的关键场景、地点状态或场景锚点记录
- `story-state` 维护的动态 JSON 状态
- `outline-manager` 中的全书纲、卷纲、角色弧线和伏笔安排
- `chapter-planner` 的章节目标、场景清单、场景节拍和章末钩子
- 最近一次更新的大纲或设定变更内容
- 用户指定的规范、禁用项或必须保留的信息

## 输出要求

- 表达直接，方便执行。
- 发现项优先按严重程度排序。
- 清楚指出冲突或风险的事实来源。
- 每个问题都要说明影响和建议修法。
- 区分已确认问题、待确认歧义和可优化提醒。
- 场景问题优先写清“哪一个场景状态没有被维护”以及“会影响哪些后续场景或章节”。
- 上下文不足时明确说明假设。

## 默认输出结构

- 严重问题：已经违背设定、规范或因果链，建议优先返修
- 高风险问题：当前文本容易引发误解、断裂、场景失忆或后续连锁问题
- 待确认项：需要用户或上游设定确认后才能定论的点
- 可优化项：不构成硬冲突，但会削弱节奏、角色可信度或伏笔效果

## 与其他 skills 的衔接

- 检查前，优先读取 `story-bible` 中的相关 canon
- 如果问题涉及重复回访场景，优先回查 `story-bible` 中的关键场景状态
- 如需对照全局结构和伏笔安排，补读 `outline-manager`
- 如需对照本章预期目标，补读 `chapter-planner`
- 大纲更新或设定更新后，也应回到本 skill 做一次变更审查
- 发现问题后，正文返修交给 `story-writer`
- 如果问题来自大纲层，回交 `outline-manager`

## 备注

- 本 skill 负责章节完成后，以及大纲或设定变更后的检测，不负责直接重写正文。
- `references/` 提供检查模板和维度约束，使用时按任务范围按需读取。
- 题材化清单用于补强通用检查，不替代基础连续性检查。
- 保持本文件精简，把详细检查模式迁移到独立参考文件。
