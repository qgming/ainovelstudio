# story-rewrite-adapt

把一部已有作品（或素材）改换目标：换平台、换赛道、换篇幅、换调性。保留可用内核，按新目标重做节奏、视角和包装。

本技能管「怎么改适配」；具体落笔仍按 `story-prose-craft`，风格调性可配合 `story-author-style`，去 AI 味用 `story-deslop`，新门面用 `story-title-blurb`。

## Use When

- 换平台：起点 / 番茄 / 晋江 / 知乎盐言等之间互改，适配目标平台节奏与审核。
- 换篇幅：长篇压成短篇（或抽某条线独立成篇）、短篇扩成长篇 / 连载。
- 换赛道 / 视角：男频改女频视角、调整目标读者群。
- 换调性：把慢热文改快节奏爽文，或反之；旧稿翻新提质。

## Inputs To Read

- 原作正文 / 大纲 / 设定：用 `workspace_read` 读被改编范围。
- `.project/README.md` 与立项文件：原作的题材、卖点、文风。
- 目标平台 / 赛道的节奏与惯例（必要时配合 `story-long-scan` / `story-short-scan` 的市场资料）。

## Procedure

1. **定改编目标**：明确从什么改到什么（源平台/篇幅/赛道/调性 → 目标）。差异大时用 `ask_user` 确认核心取舍。
2. **抽内核**：从原作提炼可保留的内核——核心冲突、人物关系、卖点、最强的几个桥段。这些是「带走」的。
3. **诊断落差**：对照目标，找出必须改的：节奏（太慢/太快）、篇幅（要砍/要补）、视角（爽点逻辑）、调性（文风）、合规（敏感点）。
4. **做改编方案**：按 `references/adapt-playbook.md` 选对应改法，先出方案（保留什么 / 砍什么 / 改什么 / 补什么），让用户确认大方向。
5. **落地执行**：按方案改写。长改短重在「砍线 + 提密度」；短改长重在「加线 + 铺垫」；换调性重在「重做节奏和句法」。
6. **配套更新**：门面（书名/简介/标签）按新平台用 `story-title-blurb` 重做；文风按目标用 `story-author-style` 校准；成稿用 `story-deslop` 收口。

## 四种改编的核心动作

详见 `references/adapt-playbook.md`：

- **换平台**：保内核，改节奏密度、开篇速度、敏感合规、门面包装。
- **长改短**：抽主线、砍支线、提爽点密度、重做独立完结的结局。
- **短改长**：拆冲突为多阶段、加支线和配角、补世界观、设连载钩子。
- **换调性 / 视角**：重做爽点逻辑（男频↔女频）、节奏和句法，内核事件可留。

## Quality Gates

- **内核留住**：原作最值钱的冲突 / 卖点 / 桥段没在改编中丢掉。
- **适配到位**：节奏、篇幅、视角、调性真正匹配目标平台 / 受众，不是原稿换皮。
- **结构自洽**：砍线后没留断头线索；加线后没注水拖节奏。
- **合规过审**：目标平台的敏感点已处理。
- **门面同步**：书名 / 简介 / 标签按新目标重做，不沿用旧门面。

## Common Failure Signals

- 只改了书名和开头，正文节奏还是原平台的，水土不服。
- 长改短时砍了支线却留下指向它的伏笔，成断头线索。
- 短改长靠注水拉长度，节奏垮掉。
- 男频改女频只换了主角性别，爽点逻辑没改（仍是「为变强不顾一切」）。

## Outputs / Write-Back

- 改编方案（保留 / 砍 / 改 / 补）：可暂存对话或写 `资料/改编方案-*.md`。
- 改编后的正文 / 大纲 / 门面：写回对应文件或新项目目录。
- 提示更新 `.project/memory/` 的项目定位（题材 / 平台 / 篇幅变化）。
