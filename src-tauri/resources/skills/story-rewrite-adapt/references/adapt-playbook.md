# 改编打法手册

改编 = 保内核 + 换适配。先想清楚「带走什么、丢掉什么、重做什么」，再动笔。

通用第一步：**抽内核**。从原作提炼出最值钱的东西——核心冲突、关键人物关系、卖点、最抓人的几个桥段。这些是跨改编带走的；其余都可重做。

---

## 一、换平台

同一作品投不同平台，内核不变，改适配层。

| 改什么 | 怎么改 |
|---|---|
| 节奏密度 | 番茄 / 七猫提速、爽点前置、压缩铺陈；起点容忍较慢的世界观铺陈；晋江重情感线密度。 |
| 开篇速度 | 免费 / 短平台开篇即冲突；订阅平台可稍缓但黄金三章仍要立住。 |
| 文风调性 | 接地气 vs 文艺，按平台读者重做句法（配合 story-author-style）。 |
| 合规 | 按目标平台审核尺度处理敏感点（暴力 / 擦边 / 价值观）。 |
| 门面 | 书名 / 简介 / 标签按目标平台惯例重做（story-title-blurb）。 |

要点：不要只改门面和开头就投，正文节奏必须整体适配，否则「水土不服」高跳出。

---

## 二、长改短（长篇 → 短篇 / 抽线成篇）

核心动作：**抽主线、砍支线、提密度、重做独立结局**。

1. **定主线**：选一条最强的冲突线 / 关系线作为短篇主干，其余线砍掉或压成背景。
2. **砍支线**：删掉与主干无关的配角线、世界观铺陈、慢热铺垫。**砍线后必查**：有没有留下指向被砍内容的伏笔 / 引用（断头线索）。
3. **提密度**：每一段都服务主冲突 / 情绪 / 反转，删一切可有可无的过场。
4. **重做结局**：短篇要独立完结、payoff 兑现，不能留长篇式开放钩子。
5. 篇幅对齐目标（短篇 1-3 万字 / 微小说千字级），用 `text_stats` 复核。

---

## 三、短改长（短篇 → 长篇 / 连载）

核心动作：**拆冲突为多阶段、加线、铺垫、设连载钩子**。最大风险是注水。

1. **拆冲突**：把短篇的单一核心冲突，拆成可分阶段升级的多个冲突节点。
2. **加支线 / 配角**：补充能独立产出爽点和情绪的支线、配角、势力，但每条线都要有功能，不为凑字。
3. **补世界观 / 设定**：短篇够用即可的设定，长篇要成体系、能支撑长线。
4. **重铺伏笔**：把短篇里一笔带过的点，铺成可长线回收的伏笔，建台账。
5. **设连载钩子**：章末钩子从「独立完结」改为「持续追读」，每章留驱动力。
6. 警惕：宁可线密、不要注水。加的内容若只是拉长度、不推进，就是垮点。

---

## 四、换调性 / 换视角

内核事件可留，重做的是「怎么讲」和「为谁讲」。

### 换调性（慢热↔快节奏 / 文艺↔爽文）
- 重做节奏：发力点位置、爽点密度、铺陈比例。
- 重做句法：长短句比、对话密度、情绪外化方式（配合 story-author-style 选目标基因）。
- 事件骨架可保留，叙述方式整体换。

### 换视角 / 赛道（男频↔女频）
- **重做爽点逻辑**：男频「为变强 / 目标不顾一切」；女频常是「关系驱动 + 情绪拉扯 + 地位反转回虐」。只换主角性别而不换爽点逻辑，是最常见的失败。
- 重做关系结构和情绪走向，匹配目标读者的代入方式。
- 卖点、世界观可部分复用，但呈现重心要转。

---

## 通用红线

- 改编自**自己的**作品或已获授权的素材；不改编他人受版权保护的作品正文。
- 抽内核 = 抽结构和卖点，不是搬他人原文。
- 改完整体过一遍连续性（可调 story-continuity-audit），确保砍 / 加之后没产生新硬伤。
