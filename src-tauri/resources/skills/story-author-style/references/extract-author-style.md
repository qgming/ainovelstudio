# 炼化作者风格：样本 → 可执行基因

把一段（或几段）样本文本，拆成可执行的「风格基因卡」。目标是让别的写作能照着这张卡复现这种味道——所以每条都要是**可判定、可执行**的指标，不是「文笔优美」这类形容词。

## 输入要求

- 至少 2-3 段代表性样本；最好覆盖**开篇、高潮、日常**三种场景，因为同一作者在不同场景的节奏不同。
- 样本来源：用户贴入、工作区文件、或 `web_read` 取的公开文本。
- 只分析「怎么写」（结构 / 节奏 / 句法 / 技法），不提取桥段、人设、设定专名——那些不可复用且涉及版权。

## 七维量化提取

逐维给出可执行结论，能量化就量化（比例、频次、字数区间）。

### 1. 句长分布
- 数一段里长句（>30字）与短句（<12字）的大致比例。
- 发力句（最短、最重的那句）出现在段落什么位置？
- 产出形如：「中长句铺陈 + 段末短句砸点，长短比约 6:4」。

### 2. 标点预算
- 逗号密度（绵密 / 规整 / 简洁）。
- 是否用破折号、省略号？感叹号频率（每段几个）？
- 产出形如：「逗号绵密，禁破折号，感叹号每段≤1」。

### 3. 动名词比
- 推进靠具体动作 + 名词，还是靠形容词 + 副词堆情绪？
- 列出标志性高频词类（如尺度名词 / 意象物象 / 设定术语）。
- 产出形如：「动作名词为主，情绪靠动作外化，少形容词」。

### 4. 对话密度
- 对话占全文比例（高 / 中 / 低）。
- 是否带「说道 / 问道」标签？对话有无来回交锋、潜台词、半句留白？
- 产出形如：「对话占比约 50%，少标签，多机锋和留白」。

### 5. 段落谱系
- 单段平均句数；长短段是否交替？独句成段的频率与用法。
- 产出形如：「段落短，2-4 句为主，转折处独句成段」。

### 6. 情绪外化
- 直接写情绪词（他很愤怒），还是用动作 / 视线 / 身体反应 / 选择外化？
- 心理描写是大段独白还是穿插在动作里？
- 产出形如：「情绪靠手部动作和视线外化，不写情绪词，无大段独白」。

### 7. 章末钩子
- 收尾习惯：悬念 / 反转 / 危险 / 选择 / 情绪裂口 / 意境留白？
- 钩子强度（强钩抛大问题 / 柔钩靠人物牵引）。
- 产出形如：「强钩，每章末抛一个更大的谜」。

## 基因卡模板

提取完按这个模板产出，写回 `.project/memory/`（`type: style`）：

```markdown
---
name: style-<作者或风格名>
description: <一句话风格定位>。Use when: 写本项目正文需要这种味道时读。
type: style
updated: <来源样本 / 日期>
---

# 风格基因：<名称>

**定位**：<一句话——是什么味道，靠什么取胜>

**七维硬指标**
1. 句长分布：<指标>
2. 标点预算：<指标>
3. 动名词比：<指标>
4. 对话密度：<指标>
5. 段落谱系：<指标>
6. 情绪外化：<指标>
7. 章末钩子：<指标>

**标志性技法**（2-3 条，可执行）
- <技法1>
- <技法2>

**慎用项**
- <这种风格的副作用 / 不适配的赛道>

**示例对照**（可选，从样本里摘一句典型，标注它体现了哪条指标）
- 样本句：「<原句>」→ 体现：<哪一维>
```

## 落地与复用

- **本项目复用**：基因卡写进 `.project/memory/`，写正文时主代理读它当风格约束。
- **跨项目复用**：用户要长期带着这套风格，调 `skill-creator`（先 `skill_read` 读其 SKILL.md），把基因卡固化成一个独立技能（如 `story-style-<名>`），description 写清「Use when: 要写出XX味道时」。
- **校准**：写完正文对照基因卡逐维自查，没命中的维度回改。重点查节奏和句法是否真的像，而不是只换了几个词。

## 红线

- 学结构 / 节奏 / 句法 / 技法，**不抄**原作桥段、人设、设定专名、标志性梗。
- 基因里不出现作者原作的专有名词。
- 风格判断带主观性，作为创作参考，不当作对作者的定论。
