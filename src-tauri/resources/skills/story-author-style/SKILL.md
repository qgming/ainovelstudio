# story-author-style

把「想写出某位作者的味道」落成可执行规则。两种能力：套用现成的作者风格基因，或现场把一段样本炼化成新基因。

本技能管「像某个特定的人写」；`story-prose-craft` 管「正文落笔的通用技法（视角 / 动作 / 对话 / 节奏 / 标点 / 人设味）」；`story-deslop` 管「成稿后清 AI 味」。三者叠用：先按本技能锁定作者基因，再按 prose-craft 落笔，最后按 deslop 收口。

风格基因是**结构、节奏、句法、技法**层面的可执行指标，不是抄桥段、人设和设定专名。学的是「怎么写」，不是「写了什么」。

## Use When

- 用户点名某位作者，要写出 / 改成那种风格（开篇、打斗、情绪、节奏、笔触）。
- 用户给一段样本（自己的或某作者的），要炼化成可复用的风格基因。
- 立项或开篇前要锁定整本的风格基调，需要一个明确的风格锚。
- 正文读着「没有那味」，要对照基因做风格校准。

## Inputs To Read

- `.project/README.md` 的写作风格约定，与 `.project/memory/` 中 `type: style` 记忆：作者既有文风优先于任何基因。
- 套用现成基因时：按男频 / 女频读 `references/style-genome-male.md` 或 `style-genome-female.md` 中对应作者条目。
- 炼化新基因时：用 `web_read` 取作者公开样本，或直接读用户贴入 / 工作区里的样本文本。

## Procedure

### A. 套用现成基因

1. 判断赛道（男频 / 女频）与目标作者，读对应基因文件里的该作者条目。
2. 抽出该基因的硬指标：句长分布、标点预算、动名词比、对话密度、段落谱系、情绪外化方式、章末钩子习惯、慎用项。
3. 写 / 改正文时，把这些指标当约束执行；与作者既有文风冲突时，以作者文风为准，只借不覆盖。
4. 写完对照基因自查：是否命中该作者的标志性节奏与句法，而非只换了几个词。

### B. 炼化新基因（样本 → 基因）

1. 拿到至少 2-3 段代表性样本（开篇 / 高潮 / 日常各一段更佳）。
2. 按 `references/extract-author-style.md` 的七维做量化提取，每维给出可执行指标而非形容词。
3. 产出一份基因卡：风格定位一句话 + 七维硬指标 + 标志性技法 2-3 条 + 慎用项。
4. 写回 `.project/memory/`（`type: style`）供本项目复用；若用户要跨项目长期复用，调 `skill-creator` 固化成独立技能（先 `skill_read` 读 skill-creator 方法论）。

## 风格基因的七个维度

炼化和套用都围绕这七维（详见 `references/extract-author-style.md`）：

1. **句长分布**：长句 / 短句比例，发力句出现在哪。
2. **标点预算**：逗号密度、是否用破折号 / 省略号、感叹号节制度。
3. **动名词比**：靠动作名词推进，还是靠形容词副词堆情绪。
4. **对话密度**：对话占比、是否带「说道」标签、有无来回交锋。
5. **段落谱系**：单段句数、长短段交替节奏、独句成段频率。
6. **情绪外化**：直接写情绪词，还是用动作 / 视线 / 身体反应外化。
7. **章末钩子**：用悬念 / 反转 / 危险 / 选择 / 情绪裂口收尾的习惯。

## Quality Gates

- **命中标志性**：套用后能看出目标作者的招牌节奏 / 句法，不是表面换词。
- **基因可执行**：每条基因是数值或可判定规则（如「对话占比 60%、禁气势形容词」），不是「气势磅礴」这类形容词。
- **作者文风优先**：与项目既有文风冲突时以作者为准，基因只补不覆盖。
- **不照抄**：没有搬目标作者的桥段、人设、设定专名。
- **慎用项落实**：注意了该作者风格的副作用（如烽火慢热不适合短篇开篇、辰东中期易注水）。

## Common Failure Signals

- 只换了几个华丽词，节奏和句法还是原样，没有真正像目标作者。
- 把作者的设定专名 / 标志桥段直接搬进正文。
- 基因写成「文笔优美、气势恢宏」这种没法执行的形容词。
- 短篇硬套慢热散文型作者，开篇就拖。

## Outputs / Write-Back

- 炼化结果写 `.project/memory/`（`type: style`）：本项目的风格基因卡。
- 跨项目复用时经 `skill-creator` 固化为独立技能。
- 套用基因改正文时写回对应正文文件（最小改动）。
- 不写：作者原作的桥段 / 设定 / 人名。
