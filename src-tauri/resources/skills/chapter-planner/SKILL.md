# chapter-planner

## 目标

- 明确章节目标、输入材料、限制条件和预期产出。
- 在规划新章节前先读懂前文推进状态、人物当前状态和相关 canon。
- 让章节规划与整体故事、人物状态和叙事语气保持一致。
- 输出简洁、可复用的章节规划结果。

## 优先使用时机

以下情况优先调用本 skill：

- 准备开始写新章节，先把这一章要完成的推进目标拆清楚
- 用户已经有前文、设定或大纲，需要先消化上下文再规划下一章
- 章节节奏失衡，需要重拆场景顺序、信息释放和冲突节拍
- 需要为 `story-writer` 先准备可执行的章节方案

如果任务还停留在全书级、卷级或阶段级结构设计，优先交给 `outline-manager`。

## 核心原则

- 新章节规划先读前文，再定本章目标。
- 只读取与当前章节强相关的设定，保持上下文聚焦。
- 章节规划服务写作执行，优先给出可落地的场景推进链。
- 先做场景清单，再展开关键场景详细规划。
- 主动场景与被动场景交替安排，保证推进与反应都成立。
- 区分已确认信息、合理推断和待确认空白，避免把猜测写成事实。

## Humanizer 核心 10 条

涉及自然语言输出时，默认遵守以下 10 条：

- 用简单词，直接说话，少解释。
- 只用基础标点 `。，！？“”`。
- 少用复杂符号和花式格式，避免分号、冒号、破折号、表情和过多加粗。
- 结构保持自然，少用机械排比、三连句和标题后空话。
- 少写空泛大词和强行升华，例如“意义”“格局”“关键时刻”。
- 少写宣传腔和夸张修饰，用具体事实替代空泛形容。
- 少用模糊归因，观点、判断和信息尽量落到明确主体。
- 多用主动表达，直接写“是”“有”“谁做了什么”。
- 删除套话和聊天痕迹，例如“希望这对你有帮助”“让我们来看看”。
- 小说场景优先演出来，用动作、对白、细节写情绪，少概述、少贴标签、少旁白解释。

## 工作流程

1. 阅读用户需求，判断当前属于新章节规划、章节重拆或补写前置规划。
2. 先读取与本章直接相关的前文，至少确认上一章结尾状态、当前冲突、人物关系位置和未回收的信息。
3. 读取相关大纲与 `story-bible` canon，优先加载人物、时间线、地点、世界规则等与本章强相关的模块。
4. 读取 `references/scene-planning-template.md`，用统一模板规划场景清单与关键场景。
5. 基于前文与设定，确定本章必须完成的推进任务：情节推进、关系变化、信息释放、悬念延续或阶段转折。
6. 先列出本章场景清单，明确每场的类型、地点、POV、核心冲突和预估字数。
7. 识别章节目标、核心冲突、关键转折、高潮位置和结尾钩子。
8. 按需为关键场景展开详细规划：
   - 主动场景使用“目标 -> 冲突 -> 挫折”
   - 被动场景使用“反应 -> 困难 -> 决定”
9. 检查节奏、连续性、人物动机、场景类型分布和前置依赖是否闭合。
10. 输出可直接交给 `story-writer` 使用的章节方案，并标出假设与待确认项。

## 创作前必读上下文

在规划新章节前，优先按以下顺序读取：

1. 当前章节前的直接前文
2. 与本章相关的章节摘要、卷纲或阶段纲
3. `story-bible` 中与本章相关的设定文件

前文读取重点：

- 上一章结尾的局面变化
- 当前人物目标、情绪和关系状态
- 正在推进的冲突、悬念和未兑现承诺
- 已经释放和暂未释放的信息
- 本章承接后最容易断裂的因果链

设定读取重点：

- 人物身份、动机、禁区和关系
- 时间线位置与事件先后
- 地点条件、移动成本和场景限制
- 世界规则、能力边界、制度约束和术语写法

如果项目已经有本地设定文件，优先读取项目中的 canon；如果设定仍由 `story-bible` 持续维护，先读取对应模块再规划本章。

## 强制参考资料

使用本 skill 规划场景时，必须读取：

- `references/scene-planning-template.md`

读取后优先吸收以下约束：

- 场景先分主动 / 被动两类
- 先列场景清单，再填关键场景详细模板
- 主动场景遵循“目标 -> 冲突 -> 挫折”
- 被动场景遵循“反应 -> 困难 -> 决定”
- 章节节奏优先检查主动 / 被动交替、冲突递增和信息释放

## 默认输出结构

规划单章时，优先整理以下部分：

- 本章作用：这一章在主线、支线或人物线里的任务
- 章节目标：主角或核心人物这一章要达成什么
- 前置状态：从哪一段前文状态接入
- 场景清单：按顺序列出场景名、地点、POV、人物、核心冲突、类型和字数
- 场景节拍：按顺序列出场景目标、冲突、推进和结果
- 关键场景详细规划：至少展开高潮场景和承上启下场景
- 信息释放：这一章透露什么，保留什么
- 情绪与节奏：每段的大致强弱和节奏变化
- 章末钩子：下一章的悬念、反转或推动点
- 风险点：容易与前文或设定打架的地方

## 需要检查的输入

- 用户说明
- 相关前文章节、章节摘要或已有正文
- `outline-manager` 维护的大纲、卷纲或阶段纲
- `story-bible` 管理的人物、时间线、地点和世界设定
- 人物动机、关系状态和当前冲突
- 时间线和连续性约束

## 输出要求

- 表达直接，方便执行。
- 优先体现“前文如何承接到本章”以及“本章如何推到下一章”。
- 保持既有故事逻辑和语气一致。
- 默认先给场景清单，再给关键场景详细规划。
- 明确每个场景属于主动还是被动，并写清核心冲突或核心困难。
- 输出结果默认应能直接交给 `story-writer` 使用。
- 上下文不足时明确说明假设。

## 与其他 skills 的衔接

- 全局结构、卷级推进和角色弧线，优先读取 `outline-manager`
- 人物、地点、世界规则和时间线，优先读取 `story-bible`
- 章节方案确定后，交给 `story-writer`
- 写完章节或调整后续内容时，交给 `continuity-check`

## 备注

- 这个 skill 的价值在于先读上下文再规划，减少“会写节拍但接不上前文”的问题。
- `references/scene-planning-template.md` 是本 skill 的默认场景模板来源。
