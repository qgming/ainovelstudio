# 书名 / 简介 / 标签公式库

门面的目标只有一个：让目标读者在 3 秒内决定「点进去」。书名负责拦截，简介负责说服，标签负责被检索到。

## 一、书名公式

网文书名不是文学命名，是流量入口。核心：**让读者一眼看出题材 + 爽点 + 是不是给自己看的**。

### 常用结构

| 公式 | 形态 | 适用 | 例式（结构示意，非具体书名） |
|---|---|---|---|
| 卖点直给 | 金手指 / 身份 + 动作 | 男频爽文、番茄向 | 「我的XX能XX」「开局一个XX」 |
| 身份反差 | 低身份 + 高能力 / 逆袭 | 升级流、打脸文 | 「废柴的我竟然XX」「被赶出家门后我XX」 |
| 关系 + 情绪 | 人物关系 + 情绪走向 | 女频言情 | 「和XX的第N次XX」「错把XX当XX」 |
| 悬念钩子 | 反常设定 / 谜 | 悬疑、设定流 | 「每天醒来都XX」「这个世界有点不对」 |
| 梗 / 网感 | 流行句式 + 题材 | 短篇、新媒体 | 口语化、自带情绪和冲突感 |
| 人设标签 | 主角性格 / 身份名片 | 角色向 | 突出主角最抓人的那个标签 |

### 命名硬规则

- **可识别**：看名字就知道男频 / 女频、什么题材、什么爽点。太雅太泛 = 自杀。
- **有动势**：动词 / 冲突 / 反差 > 静态名词堆砌。
- **平台适配**：番茄 / 七猫偏直白卖点；起点容忍中二有气势；晋江 / 女频可含蓄但要点情绪；知乎盐言短篇偏疑问句 / 第一人称代入。
- **过审安全**：避敏感词、避擦边、避明显侵权撞名。
- **长度**：移动端列表显示有限，控制在能完整显示的字数内。

### 产出方式

一次给 5-8 个候选，**覆盖不同钩子方向**（别全是卖点直给），每个标注：主打钩子 / 适配平台 / 风险。最后给一个推荐 + 理由。

## 二、简介公式

简介是第二道闸。第一句必须是钩子，**禁止用世界观 / 背景介绍开场**。

### 按赛道选结构

**男频**（卖点 + 金手指 + 爽感预期）
```
[钩子句：主角的困境或一个反常事件]
[转折：获得了什么金手指 / 身份 / 机会]
[预期：将带来什么爽感 / 走向什么高度]
```

**女频言情**（关系 + 情绪 + 冲突预期）
```
[钩子句：一段关系的裂缝 / 反常 / 强情绪事件]
[关系结构：谁和谁、什么羁绊]
[情绪预期：甜 / 虐 / 反转 / 救赎，吊住但不剧透]
```

**短篇 / 新媒体**（钩子 + 反转预告，更短更冲）
```
[强钩子：一句话甩出最冲的矛盾或悬念]
[反转预告：暗示有翻盘 / 反转，但不说是什么]
```

### 简介硬规则

- 第一句 = 钩子（冲突 / 悬念 / 强卖点），不是「在一个XX的世界里」。
- 吊预期，**不剧透**关键反转和结局。
- 给 1-3 个版本（强卖点版 / 悬念版 / 情绪版）让用户选。
- 控制长度，移动端要能不点「展开」就读到核心钩子。
- 语气贴合书名和正文文风，三者一致。

## 三、标签 / 分类

- 用**平台真实分类词**，不自造（必要时 `web_search` 查平台分类页）。
- 覆盖维度：题材 + 人设 + 情绪 + 卖点（如「都市 / 重生 / 复仇 / 打脸 / 爽文」）。
- 精准命中目标读者的检索习惯词，**不堆无关热词**——标签和内容不符会拉高跳出、伤推荐权重。
- 主标签放最能定义作品的那个，次标签补充细分人群。

## 红线

- 不抄对标书的书名 / 简介原文，只借结构和钩子方式。
- 不用敏感、擦边、侵权词。
- 简介不剧透、不虚假承诺（写了「爽文」就得真爽）。
