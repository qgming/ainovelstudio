# story-title-blurb

把一部作品包装成「门面」：书名、简介、标签、封面文案。门面决定读者点不点进来，是平台流量的第一道闸。

本技能管「怎么包装」；题材方向和卖点来自 `story-long-scan` / `story-short-scan`（扫榜）和立项文件。

## Use When

- 用户要起 / 改书名，想多个候选，或评估一个名字行不行。
- 用户要写 / 改简介、封面文案、一句话卖点。
- 用户要取平台标签 / 分类。
- 作品上架、投稿、换平台前做整体门面包装。

## Inputs To Read

- 立项文件 / `.project/README.md`：题材、卖点、目标平台、目标读者、情绪基调。
- `.project/memory/` 的 `type: project`：当前作品定位。
- 已有 `资料/市场/scan-*.md`（若有）：对标书的命名 / 标签风向。
- 目标平台的命名与分类惯例（必要时 `web_search` 查平台分类页）。

## Procedure

### 起书名

1. 锁定题材标签词、核心卖点、目标读者、平台调性。
2. 按 `references/title-blurb-formulas.md` 的命名公式产出 5-8 个候选，覆盖不同方向（卖点直给 / 悬念 / 人设 / 反差 / 梗）。
3. 每个候选标注：主打什么钩子、适配什么平台、风险（是否撞名 / 是否过审风险 / 是否太泛）。
4. 给一个推荐项 + 理由，让用户选。

### 写简介

1. 判断平台简介逻辑：男频重「卖点 + 金手指 + 爽感预期」；女频重「关系 + 情绪 + 冲突预期」；短篇重「钩子 + 反转预告」。
2. 按公式写 1-3 个版本（强卖点版 / 悬念版 / 情绪版）。
3. 开头一句必须是钩子，不用世界观背景开场；结尾留期待，不剧透反转。

### 取标签

1. 用平台真实分类词（不自造），覆盖题材 + 人设 + 情绪 + 卖点维度。
2. 标签要精准命中目标读者的检索词，不堆砌无关热词。

## Quality Gates

- **书名可点击**：3 秒内能让目标读者知道「这是给我看的、有什么爽点」。
- **简介有钩子**：第一句是冲突 / 悬念 / 卖点，不是背景介绍。
- **不剧透**：简介吊预期，不抖关键反转和结局。
- **过审安全**：书名 / 简介无敏感违规词，符合平台调性。
- **标签精准**：用平台真实分类，命中检索，不堆无关词。

## Common Failure Signals

- 书名太泛太雅，看不出题材和爽点。
- 简介在介绍世界观 / 背景，半天不进冲突。
- 简介把反转和结局剧透了。
- 标签堆砌热词但和内容不符，误导读者导致跳出。

## Outputs / Write-Back

- 写回立项 / 定位文件或 `设定/作品定位.md` 的书名 / 简介 / 标签字段。
- 候选清单可暂存对话供用户选，定稿后写回文件。
- 不写：正文、大纲、设定细节。
