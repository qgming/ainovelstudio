# outline-manager

## 目标

- 承接 `plot-planner` 已确认的 premise 和方向，把它们扩展成可持续维护的大纲体系。
- 在工作区中读取、创建和更新多层级大纲文件，保证结构资产长期可用。
- 当正文、设定或创作决策发生变化时，把变更同步回卷纲、角色弧线和伏笔回收计划。
- 检查结构层问题，包括伏笔遗漏、payoff 断裂、支线失衡、节奏塌陷和推进链断点。
- 输出可继续交给 `chapter-planner`、`story-writer` 和 `continuity-check` 的大纲结果。

## 优先使用时机

以下情况优先调用本 skill：

- 用户已经有 premise、主线方向或初版大纲，需要继续扩展
- 用户想维护全书纲、卷纲、阶段纲或角色弧线
- 用户希望把新正文、新设定或新决策同步回本地大纲文档
- 用户需要检查伏笔、回收、支线并行和节奏断点
- 用户希望把“大纲修订”作为一个独立高频动作持续执行

如果任务仍处于冷启动阶段，优先交给 `plot-planner`。

## 工作流程

1. 阅读用户需求，判断当前属于大纲创建、层级扩展、修订同步或结构审计。
2. 读取当前可用上下文：`plot-planner` 结果、`story-bible` canon、现有大纲、正文增量和用户新增约束。
3. 识别这次任务涉及哪些层级：全书、卷、阶段、章节前置、角色弧线、伏笔表、支线表。
4. 先在工作区中寻找已有大纲目录与文件；如果没有，再创建适合项目规模的结构。
5. 把已确认信息写入对应层级，区分硬事实、暂定方案和待确认项。
6. 当正文或设定变化影响结构时，同步更新相关卷纲、角色线、伏笔回收点和时间顺序。
7. 检查因果链、升级链、支线位置、钩子分布和 payoff 是否对齐。
8. 输出更新后的大纲结果，并说明下一步更适合进入哪个 skill。

## 默认输出结构

根据任务范围，优先整理以下部分：

- 全书纲：起点、主线、阶段目标、终局方向
- 卷纲：每卷目标、冲突、转折、收束点、卷尾钩子
- 阶段纲：若干章节组成的推进链、阶段内升级和信息释放
- 角色弧线：主要人物的起点、变化节点、关系变化和阶段状态
- 伏笔与 payoff：埋点位置、回收时机、回收方式、未回收项
- 支线推进表：支线目的、与主线关系、推进节点、汇合点
- 结构审计结果：节奏风险、断点、冗余段和需要补强的位置

## 需要检查的输入

- 用户说明
- `plot-planner` 产出的方向、premise 和选题结论
- `story-bible` 管理的 canon
- 现有全书纲、卷纲、章节纲或角色线笔记
- 已写正文、修订计划和新增设定
- 篇幅、卷数、更新节奏和结尾约束

## 本地维护职责

本 skill 负责维护项目工作区中的大纲资产：

- 优先复用项目已有的大纲目录、命名方式和文档风格
- 如果项目还没有大纲目录，可以创建如 `大纲/`、`outline/`、`docs/outline/`
- 根据项目规模决定聚合或拆分，不强制固定结构
- 长篇或连载项目优先拆成多层级文件，便于持续更新

## 文件结构策略

根据项目规模选择合适结构：

- 轻量项目
  - 适合 1 到 2 个聚合文件
  - 例子：`故事大纲.md`、`角色弧线.md`
- 常规项目
  - 适合按全书纲、卷纲、角色线拆分
  - 例子：`全书纲.md`、`卷纲.md`、`角色弧线.md`
- 长篇或连载项目
  - 适合按层级持续拆分
  - 例子：`全书纲.md`、`卷一大纲.md`、`卷二大纲.md`、`伏笔回收表.md`、`支线推进表.md`

## 输出要求

- 表达直接，方便继续执行。
- 始终区分已确认结构、待确认方案和风险点。
- 修订任务优先展示变更影响，而不是重写全部大纲。
- 审计任务优先指出结构问题、影响范围和建议修复点。
- 输出结果要能直接交给 `chapter-planner` 消费。

## 与其他 skills 的衔接

- 冷启动和选题阶段，先交给 `plot-planner`
- canon 建档与设定维护，交给 `story-bible`
- 单章拆解和场景节拍，交给 `chapter-planner`
- 需要写正文时，交给 `story-writer`
- 大纲和正文成型后，交给 `continuity-check`

## 备注

- 这个 skill 负责大纲体系的长期维护，不负责冷启动选题。
- 它优先处理“已经确定要写什么”之后的结构组织与修订同步。
- 保持本文件精简，后续如需题材特化模板，可放进 `references/`。
