# story-short-scan

扫一轮短篇平台风向，给出可推荐题材方向与对标短篇候选。

## Use When

- 用户要看知乎盐言 / 七猫 / 黑岩 / 点众等平台的短篇热门。
- 短篇方向的批量市场扫描子任务。

## Inputs To Read

- 用户主提示词（目标平台、目标读者、自身擅长、禁区）。
- `资料/市场/` 已有短篇调研；没有该目录时按需创建。

## Procedure

1. 用 `web_search` 拿短篇平台热门链接（知乎盐言、七猫短篇榜、黑岩故事推荐等）。
2. 用 `web_read` 读热门短篇与编辑推荐。
3. 整理：
   - 平台风向摘要
   - 可推荐题材方向（含 平台 + 卖点 + 对标短篇候选）
   - 不推荐方向 + 原因
   - 差异化建议
4. 写入 `资料/市场/scan-{时间戳或编号}.md`。

## Analysis Dimensions

- **平台差异**：知乎盐言重情绪与反转，七猫 / 点众更看强冲突和完读，黑岩偏强类型与强刺激。
- **题材结构**：统计追妻、重生复仇、死后视角、小三、世情、悬疑、职场等方向。
- **情绪模型**：看开头压迫、中段拉扯、反转爆点、结尾满足感。
- **可复制点**：只提炼结构、节奏和包装方式，不照搬具体桥段。
- **避坑**：标出过度同质化、伦理风险、平台不适配、素材门槛高的方向。

## Report Template

- 扫榜时间与来源。
- 平台风向一句话。
- 热门短篇方向表：题材 / 情绪承诺 / 代表篇 / 风险。
- 推荐方向 2-4 个：卖点、反转核心、对标篇、差异化做法。
- 不推荐方向 1-3 个。
- 下一步：建议拆哪 1-2 篇对标短篇。

## Outputs / Write-Back

- `资料/市场/scan-*.md`

## Evidence Rules

- 引用平台数据时标抓取时间与来源链接。
- 趋势判断需要至少 2 条证据；只有 1 条时标「初步信号」。
- 推荐方向的对标短篇必须含篇名 + 平台 + 一句话定位。
