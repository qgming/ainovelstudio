# 怎么写 description 让技能触发更准

`description` 是模型在「资源目录」里看到的唯一摘要，决定它要不要 `skill_read` 读完整 SKILL.md。写不好 → 该用的不用、不该用的乱用。

## 公式

```
<一句话定位>。
Use when: <场景1> / <场景2> / <场景3>，或<某子任务>时调用。
触发方式：/<英文命令>、/<中文别名>、「<口语触发词>」「<口语触发词>」
```

三段缺一不可：

1. **一句话定位**：这个技能「是什么 + 解决什么」。给类别（写 / 改 / 查 / 调研）和对象（长篇 / 短篇 / 正文 / 风格…）。
2. **Use when:**：列出该触发的具体场景，用 `/` 分隔，覆盖正常说法和子任务说法。让模型能直接判定「当前任务命不命中」。
3. **触发方式：**：列斜杠命令（英文 id + 中文别名）和用户可能说出口的口语触发词，用「」括起来。

## 正例（对标现有技能）

```
长篇网文写作的执行手册。覆盖长篇开书、卷纲、细纲、人物 / 世界观、章节正文与连续推进。
Use when: 用户要开长篇 / 写长篇大纲 / 写长篇正文 / 续写章节 / 推进连载，或长篇作者步骤执行任务时调用。
触发方式：/story-long-write、/写长篇、「帮我开书」「写大纲」「续写下一章」
```

为什么好：定位给了类别（写）和对象（长篇）；Use when 把开书 / 大纲 / 正文 / 续写 / 连载都列全，子任务说法也覆盖；触发方式同时给命令和口语。

## 反例与修法

| 反例 | 问题 | 修法 |
|---|---|---|
| `一个很强的网文写作技能。` | 没说类别 / 对象 / 触发，模型无法判定何时用 | 补全三段公式 |
| 只有定位，没有 `Use when:` | 模型不知道何时该读它，容易漏触发 | 必须加 `Use when:` 列场景 |
| `Use when: 写作时` | 太宽，会和别的写作技能抢，导致误触发 | 收窄到具体场景：长篇 / 短篇 / 正文段落 / 风格调校 |
| 触发词只有英文命令 | 用户用中文口语时触发不到 | 补中文别名和「口语触发词」 |

## 收窄 vs 放宽

- **该用没用**（漏触发）：`Use when:` 场景太少或太抽象 → 增列具体场景、补口语触发词。
- **不该用乱用**（误触发）：场景太宽或和别的技能重叠 → 收窄措辞，点明边界（如「仅短篇」「仅终稿润色」）。
- 改触发优先动 description，不动正文。改完可让用户用几句典型话术验证是否命中。
