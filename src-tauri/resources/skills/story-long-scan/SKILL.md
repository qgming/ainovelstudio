# story-long-scan

扫一轮长篇榜单，给出可推荐题材方向与对标书候选。

## Use When

- 用户要看起点 / 番茄 / 晋江 / 七猫等平台的长篇榜单与风向。
- 长篇方向的批量市场扫描子任务。

## Inputs To Read

- 用户主提示词（目标平台、目标读者、自身擅长、禁区）。
- `资料/市场/` 下已有调研：避免重复扫同一方向；没有该目录时按需创建。

## Procedure

1. 用 `web_search` 拿榜单链接（起点月票榜 / 番茄推荐榜 / 晋江总分榜等）。
2. 用 `web_read` 读榜单页与若干代表书的简介 / 标签 / 评论。
3. 整理：
   - 平台风向摘要
   - 可推荐题材方向（含 平台 + 卖点 + 对标书候选）
   - 不推荐方向 + 原因
   - 差异化建议
4. 写入 `资料/市场/scan-{时间戳或编号}.md`。

## Analysis Dimensions

- **平台指标**：起点看追读 / 月票 / 订阅信号；番茄看完读 / 留存 / 推荐曝光；晋江看收藏 / 积分 / 读者反馈。
- **题材分布**：统计榜单中重复出现的题材、关键词、金手指、关系结构。
- **新信号**：最近上升但样本仍少的方向标为「初步信号」。
- **可写性**：判断作者能力、资料门槛、更新压力、差异化空间。
- **风险**：监管风险、平台疲劳、同质化严重、素材门槛高。

## Report Template

- 扫榜时间与来源链接。
- 平台风向一句话。
- 热门题材表：题材 / 代表作 / 证据 / 趋势 / 可写性。
- 推荐方向 2-4 个：平台、卖点、对标书、差异化入口。
- 不推荐方向 1-3 个：原因与替代方案。
- 下一步：建议拆哪 1-2 本对标书。

## Outputs / Write-Back

- `资料/市场/scan-*.md`

不要写：正文、大纲、设定、选题简报（简报由后续拆解 / 简报子任务产出）。

## Evidence Rules

- 引用平台数据时标抓取时间与来源链接。
- 趋势判断需要至少 2 条证据；只有 1 条时标「初步信号」。
- 推荐方向的对标书必须含书名 + 平台 + 一句话定位。
